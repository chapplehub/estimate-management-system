# タスク完了時チェックリスト

## 必須実行コマンド

### 1. リント
```bash
npm run lint
```
- ESLint エラーがないことを確認
- 警告も可能な限り解消

### 2. テスト
```bash
npm test
```
- すべてのテストがパスすることを確認
- 新機能には必ずテストを追加

### 3. ビルド確認（必要に応じて）
```bash
npm run build
```
- TypeScript コンパイルエラーがないことを確認

## コードレビューチェックリスト

### アーキテクチャ
- [ ] レイヤー間の依存関係は正しいか
- [ ] Domain 層が外部ライブラリに依存していないか
- [ ] 責務が適切なレイヤーに配置されているか

### コード品質
- [ ] 命名規則に従っているか
- [ ] 関数が 50 行以内に収まっているか
- [ ] DRY 原則に従っているか

### 型安全性
- [ ] any 型を使用していないか
- [ ] 適切な型アノテーションがあるか
- [ ] null/undefined の扱いが適切か

### エラーハンドリング
- [ ] 適切なエラークラスを使用しているか
- [ ] エラーメッセージは明確か

### テスト
- [ ] ユニットテストが書かれているか
- [ ] 正常系・異常系・境界値をテストしているか
- [ ] Domain 層は 90%+ カバレッジ目標

### セキュリティ
- [ ] 機密情報がログに出力されていないか
- [ ] 認証・認可チェックが適切か

## テストカバレッジ目標

| レイヤー | 目標 |
|---------|------|
| Domain 層 | 90%以上 |
| Application 層 | 80%以上 |
| Infrastructure 層 | 70%以上 |

## テストデータ戦略

| 対象 | データソース |
|------|-------------|
| Value Object / Entity | インメモリ |
| Command/Query | モック |
| Repository 実装 | 実DB |
| 統合テスト / E2E | 実DB |
