# プロンプト内容

あなたは GitHub Issue 作成の専門家です。
#$ARGUMENTS について GitHub issue を作成してください。

## ステップ 1: タイプ自動判定

まず `.claude/references/commit-types.md` を読み、マッピングテーブルを取得する。
`$ARGUMENTS` のキーワードとテーブルの `keywords` 列を照合して issue タイプを推定する。
テーブルの `prefix` 列をタイトル接頭辞に、`label` 列を GitHub ラベルに使用する。

- 複数マッチした場合は最も適切なものを選択
- どれにも該当しない場合は `feat`（`Type: enhancement`）をデフォルトとする

## ステップ 2: ラベル存在チェック

`gh label list --search "Type: xxx"` でラベルの存在を確認する。
ラベルが存在しない場合はラベルなしで作成する（確認不要）。

## ステップ 3: 優先度判定（オプション）

以下のキーワードがあれば優先度ラベルも付与する:

- 「緊急」「critical」「本番障害」「ブロッカー」→ `Priority: critical`
- 「低優先」「後回し」「いつか」→ `Priority: low`
- 上記以外 → 優先度ラベルなし（付与しない）

優先度ラベルも存在チェックを行う。

## ステップ 4: テンプレート選択

判定したタイプに応じて以下のテンプレートを使用する。

### feat（enhancement）テンプレート

```markdown
## 概要

(何を実装するか)

## 背景・コンテキスト

(なぜこの機能が必要か)

## 実装タスク

- [ ] タスク1
- [ ] タスク2

## 受け入れ条件

- [ ] 条件1
- [ ] 条件2

## 関連 issue

- なし
```

### bug テンプレート

```markdown
## 症状

(何が起きているか)

## 再現手順

1. 手順1
2. 手順2

## 原因

(判明していれば記載)

## 影響範囲

(どの機能・ユーザーに影響があるか)

## 解決策

(決定した解決策 - 後で追記可)

## 関連 issue

- なし
```

### refactor テンプレート

```markdown
## 背景

(なぜリファクタリングが必要か)

## 対象

- 対象ファイル・モジュール

## 作業内容

- [ ] 作業1
- [ ] 作業2

## 完了条件

- [ ] 条件1
- [ ] 条件2

## 関連 issue

- なし
```

### test テンプレート

```markdown
## 概要

(何のテストを書くか)

## 対象

- 対象ファイル・モジュール

## テスト方針

(どのようなテストケースを書くか)

## 作業内容

- [ ] 作業1
- [ ] 作業2

## 完了条件

- [ ] 条件1
- [ ] 条件2

## 関連 issue

- なし
```

### ci テンプレート

```markdown
## 概要

(何を改善・追加するか)

## 背景

(なぜこの変更が必要か)

## 作業内容

- [ ] 作業1
- [ ] 作業2

## 完了条件

- [ ] 条件1
- [ ] 条件2

## 関連 issue

- なし
```

### docs / question / architecture テンプレート

```markdown
## 疑問・課題

(何が課題か、何を決める必要があるか)

## 背景・コンテキスト

(どういう状況で出てきた疑問・課題か、関連するコードや設計)

## 調査内容

(どう調べたか・考えたか、検討した選択肢)

## 解決策

(決定した解決策 - 後で追記)
(解決策を選んだ理由、トレードオフの説明)

## 関連 issue

- なし
```

## ステップ 5: issue 作成（確認不要）

```bash
gh issue create \
  --title "{prefix}: タイトル" \
  --label "Type: xxx" \
  --body "本文"
```

- 優先度ラベルがある場合は `--label "Priority: xxx"` も追加
- `$ARGUMENTS` の内容を分析してテンプレートの各セクションを埋める
- 関連 issue があれば本文中にリンクする

## ステップ 6: 結果を返す

作成した issue の番号と URL を簡潔に返す。
