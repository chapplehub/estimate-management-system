# 見積管理 (Estimate Management)

見積（Estimate）の作成・編集・複製・受注化を扱う業務コンテキスト。本ファイルは用語集であり、実装の詳細は含めない。

## Language

### 取引先

**取引先**:
得意先・納品先を包む上位の呼称。共通属性（コード・名称・住所・連絡先・有効フラグ）を持つ概念上のくくりであり、**独立した集約ではない**。永続化上もサブタイプ平坦化により専用テーブルを持たない（ADR-0043）。
_Avoid_: Company（コード中の `companies` 由来の語。集約としては存在しない）

**得意先**:
見積を発行する相手となる取引先。固有属性として標準マージン率（margin rate）を持つ。それ自体が集約ルート。
_Avoid_: 顧客、客先、Client

**納品先**:
得意先に紐づく、商品の届け先となる取引先。固有属性として配送時の注意事項を持つ。得意先を親に持つが、それ自体が独立した集約ルート。
_Avoid_: 配送先、届け先、Shipping address

**取引先コード**:
得意先・納品先を人が識別するための手入力コード（例: 得意先 `C001`、納品先 `D001`）。一意性は型内（得意先どうし／納品先どうし）でのみ保証され、型を跨いだ重複は許容する（ADR-0043）。
_Avoid_: 会社コード、企業コード

**標準マージン率**:
得意先ごとに既定で適用されるマージン率。見積明細の価格計算の出発点となる得意先固有の属性。
_Avoid_: 利益率、マークアップ

### 見積の複製

**複製 (Duplication)**:
既存の見積を読み取り、選択したバリエーション群から新しい見積を新採番で生成する操作。複製元は一切変更しない（読み取りのみ）。
_Avoid_: コピー、複写

**複製元 (Source)**:
複製の入力となる既存見積、およびそのバリエーション。
_Avoid_: 元見積、オリジナル

**複製先 (Copied)**:
複製で新しく生成された見積、およびそのバリエーション。
_Avoid_: コピー先、新見積（文脈で曖昧なとき）

**バリエーション系譜 / EstimateVariationCopy (Variation Genealogy)**:
複製先バリエーションが、どの複製元バリエーションから生まれたかを記録する関連。複製先バリエーション1件につき高々1件の出自を持つ。複製元見積は系譜から導出する（本体に複製元の列は持たない）。
_Avoid_: コピー履歴、複製ログ

### 見積の改訂

**得意先改訂 (Revision for Customer)**:
同一見積内で、納品先宛バリエーションを改訂元として得意先宛の新バリエーションを生成する操作。採番は行わず、同一集約インスタンス内で完結する（複製と本質的に異なる）。改訂が1件でも存在する見積は、見積年月日・税率・税端数区分・得意先・納品先を変更できない（締切日・部署は変更できる）。
_Avoid_: 複製（集約をまたぐ別操作）、コピー

**改訂元 (Revision Source)**:
得意先改訂の入力となった納品先宛バリエーション。改訂された時点で凍結される。
_Avoid_: 元バリエーション、オリジナル

**改訂先 (Revised)**:
得意先改訂で新しく生まれた得意先宛バリエーション。
_Avoid_: 改訂版、新バリエーション（文脈で曖昧なとき）

**改訂系譜 / EstimateVariationRevision (Revision Genealogy)**:
改訂先バリエーションが、どの改訂元バリエーションから生まれたかを記録する関連。改訂先1件につき高々1件の出自を持つ。複製系譜（EstimateVariationCopy）と異なり、両端が同一集約インスタンス内にあるため**集約の内側に属する**。
_Avoid_: 改訂履歴、改訂ログ

**行構成固定 (Fixed Line Structure)**:
改訂先バリエーションが明細の追加・削除を行えないこと。改訂元との明細1:1対応を保全し、明細単位の粗利を常に計算可能に保つための制約。単価・掛率・値引・数量・メモの調整は可能（凍結より弱いガード）。
_Avoid_: 凍結（改訂元側のより強いガードと混同しない）

**凍結 (Frozen)**:
改訂元となったバリエーションが、メモ以外の編集・明細の追加削除を行えなくなること。「改訂された」という事実（改訂系譜の存在）からの帰結であり、解除という操作は存在しない。有効/無効（バリエーション状態）とは直交する概念。
_Avoid_: ロック、無効化（INACTIVE と混同しない）

### 提出区分

**提出区分 (Submission Type)**:
見積書を誰宛に発行するかの区分（得意先宛 / 納品先宛）。**バリエーション単位の属性**であり、見積全体の属性ではない（同一見積内に納品先宛と得意先宛のバリエーションが共存しうる）。得意先宛バリエーションは得意先改訂を経ずに直接作成もできる（改訂は経路のひとつ）。バリエーション作成時に確定する**不変属性**であり、宛先を後から切り替える業務操作は存在しない（得意先宛が必要なら得意先改訂または新規作成で得る）。
_Avoid_: 見積の提出区分（見積単位であるかのような言い方）、バージョン、宛先変更（存在しない操作）

### 価格

**原価 (Cost Price / costPrice)**:
商品マスタ（Product）が持つ仕入・製造コスト。販売価格ではない。粗利計算の基礎。
_Avoid_: 単価、売価

**販売単価 (Selling Unit Price)**:
商品の標準的な売値。商品マスタとは別テーブルで管理する想定（スキーマ未確定）。確定後、見積単価の初期値供給元になりうる。
_Avoid_: 定価、原価

**見積単価 (Estimate Unit Price / unitPrice)**:
見積明細に都度入力される、その案件における売単価。商品マスタからは導出せず、見積ごとに確定する。複製時はクリア（要入力）。
_Avoid_: 単価（曖昧なときは原価・販売単価と区別する）

### 商品

**周辺商品 (Peripheral Product)**:
個別商品に単方向で紐づく、本体商品と一緒に提案・販売されることが多い別売りの商品。設定先になれるのは個別商品・消耗品のみ（セット商品は不可）。見積画面で本体商品の下の行として追加できる。
_Avoid_: 関連商品、related products の直訳

**セット商品 (Set Product)**:
個別商品・消耗品をグループ化するための概念的な商品。それ自体の価格は常に 0 円で、金額はすべて構成商品の価格 × 数量から算出される。ネスト（セット商品を構成商品にすること）は不可。
_Avoid_: グループ商品、バンドル

**構成商品 (Component Product)**:
セット商品を構成する個々の商品（個別商品・消耗品のみ、数量付き）。セット商品との対応関係（構成商品と数量の組）は「セット構成」と呼ぶ。
_Avoid_: 構成品、コンポーネント

**無効化 (Deactivation)**:
商品を見積で選択できない状態にする操作。データは消えず、参照済みの見積はそのまま残る。逆操作は「有効化」。
_Avoid_: 廃止、削除（削除は行の物理削除を指す別操作）

**入れ替え (Replacement)**:
商品を無効化する際、その商品を周辺商品・構成商品として参照している他商品の参照先を、指定した別商品へ一括で付け替える操作。無効化とセットでのみ行われる。
_Avoid_: 置換、差し替え

### 見積明細

**通常明細 (Normal Line)**:
どのセット群にも属さない見積明細。数量・単価・掛率・値引から自身の金額を算出する、価格付きの末端行。周辺商品を見積に追加したものもこれに含まれる（別売りのため親に金額集約されない）。
_Avoid_: 単品明細

**セット群 (Set Group)**:
セット商品を参照する見積側のグループ見出し。自前の価格を持たず、金額は構成明細の合計、表示位置は構成明細の最小並び順から導出する。見積明細（価格付き末端行）ではない別概念。
_Avoid_: セット明細（明細＝価格付き末端行と紛らわしい）、セット親行（UI の見た目に引きずられた語）

**構成明細 (Component Line)**:
セット群に属する見積明細。選択されたセット商品の構成商品をスナップショットした価格付き末端行で、通常明細と同じ列構成・振る舞いを持つ。
_Avoid_: セット子行（UI の見た目語）、構成品

### 集約

**集約インスタンス境界 (Aggregate Instance Boundary)**:
整合性の境界。複製元見積と複製先見積は同じ Estimate 型だが別々の集約インスタンスであり、各々が独立した整合性境界を持つ。系譜は2つの集約インスタンスにまたがる関連（＝どちらの集約にも属さない）。
_Avoid_: 集約をまたぐ（bounded context をまたぐ意味と混同しやすいので、文脈を明示する）
