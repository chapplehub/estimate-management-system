# プロセスとポートの基本

作成日: 2026-03-31

## 概要

OS におけるプロセスとポートの関係、およびポートの管理の仕組み。

## プロセスとは

OS 上で動いているプログラムの実体。各プロセスには OS が PID（プロセスID）を割り振る。
同じプログラム名（例: `node`）でも PID が違えば別のプロセス。

## ポートとは

ネットワーク通信の窓口番号（0〜65535）。
マンションに例えると：

- IP アドレス = マンションの住所
- ポート番号 = 部屋番号
- プロセス = 各部屋の住人

## ポートの管理者は OS

ポートは OS（カーネル）が一元管理する共有資源。プロセスは OS に「使わせてほしい」と申請する。

### ポート取得の流れ

1. プロセスが起動する
2. プロセスが OS に `bind()` で「このポートを使わせて」と申請
3. OS が空いていれば許可し、管理表に記録
4. プロセスが `listen()` で待ち受け開始
5. クライアントが接続しにくると、OS が管理表を見て正しいプロセスにデータを届ける

### ポート番号の指名 vs おまかせ

- **指名パターン**: PostgreSQL が「5432 をくれ」→ OS が承認（設定ファイルで `port = 5432` と指定）
- **おまかせパターン**: node が「空いてる番号をくれ」→ OS が「33715 をどうぞ」と割り当て

### 同じポートは同時に2つのプロセスが使えない

先客がいるポートを別のプロセスが `bind()` しようとすると OS がエラーを返す。
`pnpm dev` で「Port 3000 is already in use」と出るのはこの仕組み。

## ポート番号の範囲と意味

| 範囲 | 名称 | 用途 |
|------|------|------|
| 0〜1023 | ウェルノウンポート | HTTP=80, HTTPS=443 など予約済み。使用に root 権限が必要 |
| 1024〜49151 | 登録済みポート | PostgreSQL=5432 など、アプリが慣例的に使う |
| 49152〜65535 | エフェメラルポート | OS が一時的に自動割り当てする範囲 |

開発時に 3000 や 8080 を使うのは、1023 以下の root 権限制約を避ける理由もある。

## 確認コマンド

```bash
# エフェメラルポートの範囲を確認
cat /proc/sys/net/ipv4/ip_local_port_range
```
