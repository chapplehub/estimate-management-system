# /dev/tty とは何か

作成日: 2026-03-27

## 概要

`/dev/tty` は現在のプロセスに紐づいている制御端末（ターミナル）を指す特殊なデバイスファイル。

## 詳細

### /dev/tty の役割

- どのターミナルから実行しても、`/dev/tty` は常に「自分のターミナル」を指すエイリアス
- 実体は環境によって異なる（`/dev/pts/0`, `/dev/pts/1` など）

```bash
# 実体の確認
$ tty
/dev/pts/0    # ターミナルごとに異なる
```

### なぜ /dev/tty を使うのか

Git hookのような場面では、stdinがGitのパイプに繋がっていてターミナルではなくなっている。具体的なデバイスパス（`/dev/pts/0`など）は環境ごとに変わるので直接指定できない。

`/dev/tty` なら環境に関係なく「今このプロセスが動いているターミナル」を指してくれるので、確実にキーボード入力を取り戻せる。

```bash
# Git hook内での使用例
exec < /dev/tty    # stdinをターミナルに繋ぎ直す
npx cz --hook      # 対話的プロンプトがキーボード入力を受け取れる
```
