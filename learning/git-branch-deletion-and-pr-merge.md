# Gitブランチ削除後もPRマージできる理由

作成日: 2026-03-17

## 概要

featureブランチを削除してしまった後でも、GitHub上のPRをスカッシュマージできた。なぜブランチを消してもマージできるのかを理解する。

## 詳細

### ブランチは「ラベル（ポインタ）」にすぎない

Gitのブランチは特定のコミットを指すポインタであり、コミットそのものではない。ブランチを削除してもラベルが剥がれるだけで、コミット自体はリポジトリに残り続ける。

```
ブランチを消す前:
  feat/issue-101 → コミットC  (ラベルが貼ってある)

ブランチを消した後:
                   コミットC  (ラベルは剥がれたが、コミット自体は残っている)
```

### ローカルブランチ削除がリモートに影響しない理由

`git push` した時点でコミットはリモート（GitHub）にコピー済み。ローカルブランチを消してもリモートの `origin/feat/issue-101` には影響しないため、PRは問題なくマージできる。

### リモートブランチを消しても大丈夫な場合がある

GitHubのPRは作成時にコミットの参照（SHA）を内部的に保持しているため、リモートブランチを削除してもPR上のコミットはしばらく残り、マージ可能な場合がある。

### コミットが本当に削除されるタイミング

`git gc`（ガベージコレクション）が実行され、どこからも参照されなくなったコミットが初めて削除対象になる。

| 操作 | 影響 |
|---|---|
| ローカルブランチ削除 | ローカルのラベルが消えるだけ。リモートに影響なし |
| リモートブランチ削除 | GitHubが参照を保持しているのでPRはマージ可能 |
| `git gc` | どこからも参照されないコミットが初めて削除される |

## 参考

- 実体験: featureブランチで開発→developにpush→PR作成→ローカルブランチ削除→スカッシュマージ成功→git pullで取り込み成功
