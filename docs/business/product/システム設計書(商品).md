## 1. システム概要

### 1.1 商品マスタの役割

商品マスタは見積・受注で使用する商品情報を管理する。

```
商品マスタ管理:
- 商品の登録・編集・無効化・削除
- 周辺商品の設定 (個別商品のみ)
- セット商品の構成管理

見積との連携:
- 見積明細で商品を選択
- 商品区分に応じた見積画面の振る舞い制御
- 価格テーブルとの紐づけ (価格機能で別途設計)
```

### 1.2 商品区分の概要

| 区分       | 説明                                       | 価格     | 周辺商品 |
| ---------- | ------------------------------------------ | -------- | -------- |
| 個別商品   | 機械やそれを構成する部品などの商品           | あり     | 設定可能 |
| 消耗品     | 個別商品ではない補助的に使用する商品         | あり     | 設定不可 |
| セット商品 | 個別商品や消耗品をまとめるための概念的な商品 | 常に 0 円 | 設定不可 |

### 1.3 権限

| 操作       | 誰ができる? |
| ---------- | ----------- |
| 商品作成   | 全従業員    |
| 商品編集   | 全従業員    |
| 商品無効化 | 全従業員    |
| 商品有効化 | 全従業員    |
| 商品削除   | 全従業員    |

※ 承認フローは不要

## 2. 商品区分

### 2.1 区分一覧 (Enum)

| 値               | 表示名     | 説明                                             |
| ---------------- | ---------- | ------------------------------------------------ |
| INDIVIDUAL       | 個別商品   | 機械や部品。周辺商品を設定できる                 |
| CONSUMABLE       | 消耗品     | 補助的に使用する商品。周辺商品は持てない         |
| SET              | セット商品 | 個別商品・消耗品をグループ化。価格は常に 0 円    |

### 2.2 区分ごとの機能制約

```
個別商品 (INDIVIDUAL):
  ✅ 周辺商品の設定
  ✅ 価格の設定
  ✅ セット商品の構成商品になれる
  ✅ 他の個別商品の周辺商品になれる

消耗品 (CONSUMABLE):
  ❌ 周辺商品の設定 (持てない)
  ✅ 価格の設定
  ✅ セット商品の構成商品になれる
  ✅ 個別商品の周辺商品になれる

セット商品 (SET):
  ❌ 周辺商品の設定 (持てない)
  ❌ 価格の設定 (常に0円)
  ❌ セット商品の構成商品になれない
  ❌ 周辺商品になれない
  ✅ 構成商品 (個別商品・消耗品) を数量付きで保持
```

## 3. 周辺商品

### 3.1 概要

- 個別商品に対して関連する商品 (個別商品 or 消耗品) を紐づける機能
- 関係は **単方向** (A→B を設定しても B→A にはならない)
- 見積画面で個別商品を選択した際、周辺商品ボタンから追加できる

### 3.2 制約

```
設定元 (周辺商品を持てる):
  ✅ 個別商品のみ

設定先 (周辺商品として設定できる):
  ✅ 個別商品
  ✅ 消耗品
  ❌ セット商品

その他:
  ❌ 自分自身を周辺商品に設定不可
  ❌ 同じ商品を重複して周辺商品に設定不可
```

### 3.3 見積画面での動作

```
1. 見積明細で個別商品Aを選択
2. 個別商品Aに周辺商品が設定されている場合、周辺商品ボタンが有効化
3. 周辺商品ボタンを押すと、周辺商品の一覧が表示される
4. 選択した周辺商品が個別商品Aの下の行に追加される
```

## 4. セット商品

### 4.1 概要

- 個別商品・消耗品をグループ化するための概念的な商品
- セット商品自体の価格は常に 0 円 (グループの役割のみ)
- 金額はすべて構成商品の価格 × 数量で算出される

### 4.2 制約

```
構成商品として設定できる:
  ✅ 個別商品
  ✅ 消耗品
  ❌ セット商品 (ネスト不可)

その他:
  ❌ 同じ商品を重複して構成商品に設定不可
  ✅ 構成商品ごとに数量 (正の整数) を指定
```

### 4.3 見積画面での動作

```
1. 見積明細でセット商品Xを選択
2. セット商品Xに紐づく構成商品が自動的にセット商品の下に追加される
3. 各構成商品の数量はセット構成テーブルの数量が初期値
4. 金額計算は構成商品それぞれの価格 × 数量の合計
5. セット割引のような概念はない
```

### 4.4 金額計算例

```
セット商品X (価格: 0円)
  ├─ 構成商品A (個別商品): 単価 100,000円 × 2台 = 200,000円
  ├─ 構成商品B (消耗品):   単価   5,000円 × 10個 =  50,000円
  └─ 構成商品C (個別商品): 単価  30,000円 × 1台 =  30,000円
                                          合計 = 280,000円
```

## 5. 商品の有効・無効管理

### 5.1 無効化の影響

```
見積作成画面:
  ❌ 無効商品は商品選択の候補に表示されない
  ❌ 無効商品を指定して見積を更新するとサーバエラー

既存データ:
  ✅ 無効化前に作成済みの見積には影響なし (過去データは維持)
  ✅ 既存の見積に含まれる無効商品はそのまま表示・参照可能

再有効化:
  ✅ 無効商品を再度有効にすることが可能
```

### 5.2 無効化時の周辺商品・セット構成への影響

```
周辺商品として設定されている場合:
  - 周辺商品の設定はそのまま残る
  - 見積画面で周辺商品ボタンを押した際、無効商品は候補に表示されない

セット構成商品として設定されている場合:
  - セット構成の設定はそのまま残る
  - 見積画面でセット商品を選択した際、無効な構成商品も追加される
    (セットの構成は変えず、無効商品が含まれる旨のワーニングを表示)
```

## 6. 商品無効化時の入れ替え機能

### 6.1 概要

商品詳細画面で商品を無効化する際、その商品が周辺商品やセット構成商品として他の商品に設定されている場合、別の商品に入れ替えることができる機能。

### 6.2 画面フロー

```
1. 商品詳細画面で無効化ボタンを押す
2. 対象商品が周辺商品テーブル・セット構成テーブルで参照されているか確認
3-a. 参照されていない場合:
     → そのまま無効化して完了
3-b. 参照されている場合:
     → 入れ替え確認ポップアップを表示

ポップアップ:
┌──────────────────────────────────────────────┐
│ 商品の入れ替え確認                            │
│                                              │
│ この商品は以下で使用されています:             │
│ ・商品A (XX-001) の周辺商品                   │
│ ・商品B (XX-002) の周辺商品                   │
│ ・セット商品C (ST-001) の構成商品              │
│                                              │
│ 入れ替え先の商品コード: [____________]        │
│                                              │
│ [入れ替えて更新]  [入れ替えずに更新]          │
└──────────────────────────────────────────────┘
```

### 6.3 入れ替えの処理

```
「入れ替えて更新」を押した場合:
  1. 対象商品を無効化 (isActive = false)
  2. 周辺商品テーブル・セット構成テーブル内の対象商品のレコードを削除
  3. 入れ替え先商品で同じ箇所に新しいレコードを追加 (数量は維持)
  ※ 1つの商品コードですべての参照箇所を一括入れ替え

「入れ替えずに更新」を押した場合:
  1. 対象商品を無効化 (isActive = false)
  2. 周辺商品テーブル・セット構成テーブルのレコードはそのまま残す
```

### 6.4 入れ替え先の制約

```
入れ替え先の商品:
  ✅ 有効な商品であること
  ✅ 周辺商品の入れ替え: セット商品でないこと (個別商品 or 消耗品)
  ✅ セット構成の入れ替え: セット商品でないこと (個別商品 or 消耗品)
  ❌ 無効な商品
  ❌ 自分自身 (無効化対象の商品)
```

### 6.5 重複チェック

```
入れ替え先の商品が、入れ替え先の周辺商品・セット構成にすでに存在する場合:
  → エラー: 「{商品名}で重複します。」

例:
  商品Aの周辺商品: [X, Y]
  Xを無効化してYに入れ替えようとした場合:
  → エラー: 「商品A (XX-001) で重複します。」
```

### 6.6 入れ替え例

```
状態:
  商品A (個別) の周辺商品: [X(数量:3), Y(数量:5)]
  セットS の構成商品: [X(数量:2), Z(数量:1)]

操作: Xを無効化 → 入れ替え先として商品Wを指定

結果:
  商品X: isActive = false
  商品A (個別) の周辺商品: [W(数量:3), Y(数量:5)]  ← X→Wに入れ替え、数量3は維持
  セットS の構成商品: [W(数量:2), Z(数量:1)]        ← X→Wに入れ替え、数量2は維持
```

## 7. 商品の削除

### 7.1 削除条件

```
物理削除が可能:
  ✅ 見積・受注で一度も使用されていない商品

物理削除が不可:
  ❌ 見積明細で使用中の商品
  ❌ 受注明細で使用中の商品
  → 無効化のみ可能
```

### 7.2 削除時の関連データ

```
商品を削除した場合:
  - 周辺商品テーブルの該当レコードも削除 (設定元・設定先の両方)
  - セット商品構成テーブルの該当レコードも削除
    (セット商品が削除された場合は構成商品レコードがすべて削除)
    (構成商品が削除された場合はその構成商品のレコードのみ削除)
```

## 8. ユースケース

### 8.1 商品作成

```
入力:
  - 商品コード (必須, 英数字, 最大50桁, ユニーク)
  - 商品区分 (必須)
  - 商品名 (必須, ユニーク)
  - 単位 (必須)
  - 商品説明 (任意)
  - 備考 (任意)

処理:
  1. 商品コードの重複チェック
  2. 商品名の重複チェック
  3. 商品を有効状態で保存

出力:
  - 作成された商品
```

### 8.2 商品編集

```
入力:
  - 商品ID
  - 変更項目 (商品コード, 商品名, 単位, 商品説明, 備考)

処理:
  1. 商品の存在確認
  2. 商品コード変更時: 重複チェック
  3. 商品名変更時: 重複チェック
  4. 商品を更新

制約:
  - 商品区分は変更不可
  - 有効・無効に関わらず編集可能

出力:
  - 更新された商品
```

### 8.3 商品無効化

```
入力:
  - 商品ID

処理:
  1. 商品の存在確認
  2. すでに無効の場合はエラー
  3. 有効フラグを false に設定

出力:
  - 無効化された商品
```

### 8.4 商品有効化

```
入力:
  - 商品ID

処理:
  1. 商品の存在確認
  2. すでに有効の場合はエラー
  3. 有効フラグを true に設定

出力:
  - 有効化された商品
```

### 8.5 商品削除

```
入力:
  - 商品ID

処理:
  1. 商品の存在確認
  2. 見積・受注で使用中の場合はエラー
  3. 関連する周辺商品レコードを削除
  4. 関連するセット商品構成レコードを削除
  5. 商品を物理削除

出力:
  - なし (成功)
```

### 8.6 周辺商品設定

```
入力:
  - 商品ID (設定元: 個別商品のみ)
  - 周辺商品リスト [{ 周辺商品ID, 数量 }] (個別商品 or 消耗品)

処理:
  1. 設定元商品の存在確認・区分チェック (個別商品か)
  2. 各周辺商品の存在確認・区分チェック (セット商品でないか)
  3. 自分自身が含まれていないかチェック
  4. 数量が正の整数かチェック
  5. 周辺商品を一括更新 (既存を削除して再設定)

出力:
  - 更新された周辺商品一覧
```

### 8.7 セット商品構成設定

```
入力:
  - セット商品ID
  - 構成商品リスト [{ 構成商品ID, 数量 }]

処理:
  1. セット商品の存在確認・区分チェック (セット商品か)
  2. 各構成商品の存在確認・区分チェック (セット商品でないか)
  3. 数量が正の整数かチェック
  4. 構成商品を一括更新 (既存を削除して再設定)

出力:
  - 更新されたセット構成一覧
```

### 8.8 商品一覧検索

```
検索条件:
  - 商品コード (部分一致)
  - 商品名 (部分一致)
  - 商品区分 (完全一致)
  - 有効フラグ (完全一致)

ソート:
  - 商品コード昇順 (デフォルト)

ページネーション:
  - オフセットベース
```

### 8.9 商品無効化 (入れ替えあり)

```
入力:
  - 商品ID (無効化対象)
  - 入れ替え��商品コード

処理:
  1. 無効化対象商品の存在確認
  2. すでに無効の場合はエラー
  3. 入れ替え先商品の存在確認 (商品コードで検索)
  4. 入れ替え先商品の制約チェック:
     - 有効な商品であること
     - セット商品でないこと (周辺商品・セット構成の入れ替え先として不適格)
  5. 重複チェック:
     - 無効化対象が参照されている各箇所について
       入れ替え先がすでに存在する場合はエラー
       (「{商品名} ({商品コード}) で重複します。」)
  6. 無効化対象商品を無効化 (isActive = false)
  7. 周辺商品テーブル・セット構成テーブルの入れ替え:
     - 無効化対象の参照レコードを削除
     - 入れ替え先で同じ箇所に新規レコード��作成 (数量は維持)
  ※ トランザクション内で一括実行

出力:
  - 無効化された商品
  - 入れ替え結果サマリ (入れ替えた箇所の件数)
```

## 10. データベーススキーマ

### 10.1 商品テーブル

```prisma
// 商品区分
enum ProductCategory {
  INDIVIDUAL // 個別商品
  CONSUMABLE // 消耗品
  SET        // セット商品
}

// 単位
enum ProductUnit {
  UNIT  // 台
  PIECE // 個
  ROLL  // 本
  BOX   // 箱
  SHEET // 枚
  SET   // セット
}

model Product {
  id          String          @id @db.Uuid // ドメイン層でUUIDv7生成
  code        String          @unique      // 商品コード (英数字, 最大50桁)
  name        String          @unique      // 商品名
  category    ProductCategory              // 商品区分
  description String?         @db.Text     // 商品説明
  note        String?         @db.Text     // 備考
  unit        ProductUnit                  // 単位
  isActive    Boolean         @default(true) @map("is_active") // 有効フラグ

  // 周辺商品 (この商品が設定元)
  relatedProducts    ProductRelation[] @relation("ProductRelationSource")
  // 周辺商品 (この商品が設定先)
  relatedByProducts  ProductRelation[] @relation("ProductRelationTarget")

  // セット商品構成 (この商品がセット商品)
  setComponents      SetProductComponent[] @relation("SetProduct")
  // セット商品構成 (この商品が構成商品)
  componentOfSets    SetProductComponent[] @relation("ComponentProduct")

  // 見積明細からの逆参照
  estimateItems      EstimateItem[]

  // 修理対象機器としての逆参照
  repairEstimates    Estimate[]

  // タイムスタンプ
  createdAt DateTime @default(now()) @map("created_at") @db.Timestamptz(3)
  updatedAt DateTime @updatedAt @map("updated_at") @db.Timestamptz(3)

  @@index([code])
  @@index([category])
  @@index([isActive])
  @@map("products")
}
```

### 10.2 周辺商品テーブル

```prisma
model ProductRelation {
  // 設定元 (個別商品のみ)
  productId        String  @map("product_id") @db.Uuid
  product          Product @relation("ProductRelationSource", fields: [productId], references: [id], onDelete: Cascade)

  // 設定先 (個別商品 or 消耗品)
  relatedProductId String  @map("related_product_id") @db.Uuid
  relatedProduct   Product @relation("ProductRelationTarget", fields: [relatedProductId], references: [id], onDelete: Cascade)

  // 数量
  quantity         Int     // 正の整数

  // タイムスタンプ
  createdAt DateTime @default(now()) @map("created_at") @db.Timestamptz(3)
  updatedAt DateTime @updatedAt @map("updated_at") @db.Timestamptz(3)
```
