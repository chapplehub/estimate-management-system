## 1. システム概要

### 1.1 業務フロー

```
1. 見積作成
   ↓
2. バリエーション作成 (複数の選択肢を作成)
   ↓
3. 客先と折衝
   ↓
4. 見積申請 (上長へ)
   ↓
5. 見積承認
   ↓
6. 受注作成
   ↓
7. 受注確定
   ↓
8. 受注票出力
```

### 1.2 見積の種類

| 見積区分 | 接頭辞     | 説明                       | 備考                           |
| -------- | ---------- | -------------------------- | ------------------------------ |
| 新規見積 | N (New)    | 新規の機械を設置する場合   | -                              |
| 修理見積 | R (Repair) | 設置済み機械の修理 (事前)  | 修理予定日が必要               |
| 事後見積 | A (After)  | 緊急対応で修理後に見積作成 | 修理実施日・緊急対応理由が必要 |

### 1.3 見積ステータス

| ステータス | 説明             | 次のアクション |
| ---------- | ---------------- | -------------- |
| 作成中     | 見積作成・編集中 | 申請           |
| 申請中     | 上長承認待ち     | 承認 / 差戻    |
| 差戻       | 修正が必要       | 再申請         |
| 取下       | 申請取り下げ     | -              |
| 承認済     | 承認完了         | 受注作成       |
| 受注作成済 | 受注作成完了     | 受注確定       |
| 受注確定済 | 受注確定完了     | -              |

## 2. 見積番号の採番ルール

### 2.1 番号形式

```
N2500001
│││└──┴─ 連番(5桁): 00001〜99999
││└───── 年度(2桁): 25 = 2025年度(2025/4〜2026/3)
│└────── 区分(1文字): N(新規) / R(修理) / A(事後)
└─────── 固定
```

### 2.2 採番仕様

- **年度**: 4 月始まり (2025 年 4 月〜2026 年 3 月 = 2025 年度)
- **リセット**: 年度ごとにリセット
- **容量**: 年間 99,999 件まで対応可能 (想定: 年間 1,000 件)
- **欠番**: 許容される (削除した見積番号は再利用しない)

### 2.3 採番タイミング

見積保存時に自動採番

### 2.4 実装例

```typescript
// 見積番号生成例
N2500001; // 2025年度の新規見積 1番
R2500123; // 2025年度の修理見積 123番
A2510500; // 2025年度の事後見積 10500番
```

## 3. 見積バリエーション

### 3.1 概要

- 1 つの見積番号に対して複数のバリエーション(選択肢)を作成可能
- 見積と見積バリエーションは 1 対多 の関係
- 客先との折衝用に複数案を用意するための機能

### 3.2 バリエーション番号

```
N2500001-01
│       └─┴─ バリエーション番号(2桁): 01〜99
└─────────── 親見積番号
```

### 3.3 バリエーションのステータス

| ステータス      | 説明           | 制約                         |
| --------------- | -------------- | ---------------------------- |
| 有効 (ACTIVE)   | 使用可能な状態 | 編集・申請可能               |
| 無効 (INACTIVE) | 使用しない状態 | 編集・申請不可、有効化は可能 |

### 3.4 バリエーションの制約

#### 申請の制約

- **1 見積番号につき申請できるバリエーションは 1 つのみ**
- 申請中の見積に対して、他のバリエーションを申請することは不可

#### 無効化の制約

- 申請中・承認済みのバリエーションは無効化不可
- 無効状態のバリエーションは編集・申請不可
- 無効 → 有効への変更は無条件で可能

### 3.5 バリエーションテーブルの項目

```
- バリエーション番号
- ステータス (有効/無効)
- 客先メモ
- 内部メモ
- 全体値引金額
- 見積明細 (1対多)
- 申請・承認情報
```

## 4. 見積から受注への流れ

### 4.1 基本フロー

```
見積承認済
  ↓
受注作成 (DRAFT)
  ↓ 納期・メモ入力
受注確定 (CONFIRMED)
  ↓
受注票出力
```

### 4.2 見積と受注の関係

- **見積番号 = 受注番号** (受注番号は別途持たない)
- **1 見積バリエーション : 1 受注** (1 対 1 の関係)
- 受注テーブルは見積テーブルを参照 (スナップショット不要)

### 4.3 受注作成の条件

```
✅ 承認済みバリエーションから作成可能
✅ 承認不要な見積も作成可能
✅ 誰でも作成可能 (権限: 作成者 + 同部署メンバー)
```

### 4.4 受注ステータス

| ステータス | 説明   | 編集可能項目 |
| ---------- | ------ | ------------ |
| DRAFT      | 作成中 | 納期、メモ   |
| CONFIRMED  | 確定   | メモのみ     |
| CANCELLED  | 取消   | 編集不可     |

### 4.5 受注確定の必須項目

```
必須:
- 納期

任意:
- メモ
```

### 4.6 受注票の出力

- **DRAFT 状態**: SAMPLE 印字で出力可能
- **CONFIRMED 状態**: 正式版を出力可能

### 4.7 受注取消

```
取消処理:
1. 受注ステータス → CANCELLED
2. 見積データを複製
3. 新バリエーション作成画面へ遷移
4. 見積ステータス → 作成中
```

### 4.8 データ整合性

- **受注作成後、見積バリエーションは編集不可**
- 受注作成時に見積ステータス: 承認済 → 受注作成済
- 受注確定時に見積ステータス: 受注作成済 → 受注確定済
- 受注は見積データを参照 (金額等の変更は不可)

## 5. 見積の複製

### 5.1 複製可能条件

- **すべてのステータスから複製可能**
- **誰でも複製可能** (他人の見積も可)
- **複製回数無制限**

### 5.2 バリエーション選択

```
選択可能:
- 複数選択可能
- 無効も含めてすべて選択可能
- 選択なしもOK (見積のみ複製)

選択順序:
- 保持される
- 連番で振り直し (01, 02, 03...)
```

### 5.3 引き継ぎ項目

#### 見積テーブル

| 項目       | 引き継ぎ | 備考               |
| ---------- | -------- | ------------------ |
| 見積区分   | ✅       | 変更可能           |
| 見積年月日 | 🔄       | 複製時の日付に更新 |
| 締切年月日 | 🔄       | 複製時から計算     |
| 提出区分   | ✅       | -                  |
| 得意先     | ✅       | -                  |
| 納品先     | ✅       | -                  |
| 消費税率   | 🔄       | 複製時点の税率     |
| メモ       | ✅       | -                  |
| 見積番号   | ❌       | 保存時に新規採番   |
| ステータス | ❌       | 作成中             |

#### バリエーションテーブル

| 項目                  | 引き継ぎ | 備考            |
| --------------------- | -------- | --------------- |
| メモ                  | ✅       | -               |
| 見積明細 (品目、数量) | ✅       | -               |
| 単価                  | ❌       | クリア (要入力) |
| 申請・承認情報        | ❌       | クリア          |
| ステータス            | ❌       | すべて有効      |

### 5.4 複製元情報の保持

```
見積テーブル:
- copiedFromEstimateId (複製元見積ID)

バリエーションテーブル:
- copiedFromVariationNumber (複製元バリエーション連番)
```

### 5.5 複製フロー

```
1. 複製ボタン
   ↓
2. バリエーション選択ダイアログ
   ↓
3. 見積新規作成画面へ遷移 (プレビュー)
   - 複製データが表示される
   - 見積区分は変更可能
   - 単価は空欄 (要入力)
   ↓
4. ユーザーが編集
   ↓
5. 保存ボタン → 見積番号採番 & DB保存
```

## 6. 修理見積と事後見積

### 6.1 見積区分の構造

```
見積区分 (EstimateType):
- NEW (新規)
- REPAIR (修理)
  └─ isAfterService: false (事前修理)
  └─ isAfterService: true (事後修理)
```

UI 上は 3 つのセレクトボックス: [新規 / 修理 / 事後]
データ上は: estimateType + isAfterService で管理

### 6.2 見積区分別の必須項目

#### 共通項目

```
- 見積年月日
- 締切年月日
- 得意先 or 納品先
- 消費税率
- 見積明細
```

#### 修理見積 (事前) のみ

```
- 修理対象機器 (商品ID)
- 故障内容
- 修理予定日
```

#### 事後見積のみ

```
- 修理対象機器 (商品ID)
- 故障内容
- 修理実施日 (過去日付)
- 緊急対応理由
```

### 6.3 事後見積の特殊制約

#### 承認フロー

```
✅ 承認不要 (すでに作業済みのため)
✅ 作成 → 即受注作成が可能
```

#### 金額チェック

```
10万円以上の場合:
- 保存時: 警告表示
- 受注作成時: 警告表示

警告内容:
「事後見積ですが、10万円を超えています。よろしいですか?」
[OK] [キャンセル]
```

#### 緊急対応理由

```
✅ 必須入力
✅ 見積書に特記事項として表示
```

### 6.4 見積書タイトル

| 見積区分 | タイトル       | 特記事項           |
| -------- | -------------- | ------------------ |
| 新規見積 | 新規設置見積書 | -                  |
| 修理見積 | 修理見積書     | -                  |
| 事後見積 | 事後修理見積書 | 緊急対応理由を表示 |

### 6.5 業務フロー比較

#### 修理見積 (事前)

```
1. 故障連絡
2. 見積作成
3. 申請 → 承認
4. 受注作成
5. 修理実施
6. 受注確定
```

#### 事後見積

```
1. 故障連絡 (緊急)
2. 修理実施 ★先に実施
3. 見積作成 (承認スキップ)
4. 受注作成 (必須)
5. 受注確定
```

## 7. 得意先と納品先の関係

### 7.1 基本構造

```
得意先 (Customer)
  └─ 納品先 (DeliveryLocation) ※1対多

取引フロー:
X社 → 得意先Y (卸値100万) → 納品先Z (販売価格120万)
```

### 7.2 提出区分による業務フロー

#### ステップ 1: 納品先見積作成

```
提出区分: 納品先
金額: 120万円 (販売価格)
制約: 申請・受注作成不可
用途: 得意先改訂の元データ
```

#### ステップ 2: 得意先改訂

```
「得意先改訂」ボタン
  ↓
納品先バリエーションを複製
  ↓
提出区分: 納品先 → 得意先
金額: 120万円 → 100万円 (調整)
粗利: 20万円を表示
```

#### ステップ 3: 申請・受注作成

```
提出区分: 得意先の見積のみ
申請・承認・受注作成が可能
```

### 7.3 見積書の出力内容

#### 納品先向け見積書

```
to: △△薬局 御中
from: 〇〇商事 (得意先)
金額: 120万円
粗利: 非表示
```

#### 得意先向け見積書

```
to: 〇〇商事 御中
from: X社
金額: 100万円
粗利: 20万円 ★表示
納品先: △△薬局 (参考表示)
```

### 7.4 受注票の内容

```
受注先: 得意先Y (〇〇商事)
納品先: 納品先Z (△△薬局)
金額: 100万円 (得意先への卸値)

※実際の納品は納品先Zへ行う
```

### 7.5 データ構造

```
見積テーブル:
- submissionType: CUSTOMER / DELIVERY_LOCATION
- customerId: 得意先ID (常に設定)
- deliveryLocationId: 納品先ID (常に設定)
- revisedFromVariationId: 得意先改訂の元バリエーションID

制約:
- 得意先選択時、配下の納品先のみ選択可能
- 両方常に設定、submissionTypeで用途を制御
```

## 8. 金額計算

### 8.1 金額計算フロー

```
明細レベル:
1. 基本金額 = 数量 × 単価 (端数切捨)
2. 掛率適用後金額 = 基本金額 × 掛率 (端数切捨)
3. 最終明細金額 = 掛率適用後金額 - 明細値引金額

見積レベル:
4. 最終明細金額小計 = Σ(最終明細金額)
5. 全体値引後金額 = 最終明細金額小計 - 全体値引金額
6. 消費税額 = 全体値引後金額 × 消費税率 (端数処理: 選択式)
7. 最終合計金額 = 全体値引後金額 + 消費税額
```

### 8.2 値引きの種類

#### ① 掛率値引 (明細レベル)

```
場所: 見積明細テーブル
計算: 単価100,000円 × 掛率0.95 = 95,000円
用途: 割引率による値引き
```

#### ② 明細値引 (明細レベル)

```
場所: 見積明細テーブル
計算: 掛率適用後95,000円 - 明細値引5,000円 = 90,000円
用途: 明細単位の値引き額
```

#### ③ 全体値引 (見積レベル)

```
場所: 見積バリエーションテーブル
計算: 明細小計1,000,000円 - 全体値引50,000円 = 950,000円
用途: 合計額からの値引き
```

### 8.3 消費税の計算

#### 計算タイミング

```
小計後に一括計算 (明細ごとではない)
```

#### 端数処理

```
見積テーブルに保持:
- taxRoundingType: ROUND_DOWN / ROUND_UP / ROUND

選択肢:
- 切り捨て (ROUND_DOWN)
- 切り上げ (ROUND_UP)
- 四捨五入 (ROUND)
```

### 8.4 粗利の計算 (得意先向け見積)

```
粗利 = 納品先価格 - 得意先価格

例:
納品先価格: 1,200,000円
得意先価格: 1,000,000円
粗利: 200,000円
粗利率: 16.7% (200,000 / 1,200,000)
```

### 8.5 金額の保存形式

```
明細テーブル:
- 税抜金額のみ保存
- 計算結果 (基本金額、掛率適用後金額、最終明細金額) を保存

見積バリエーションテーブル:
- 小計、税額、合計を保存
```

### 8.6 金額の再計算タイミング

#### 再計算を実行

```
✅ 見積保存時: 税率チェック + 金額再計算
✅ 見積申請時: 税率チェック + 金額再計算
✅ 受注作成時: 承認不要の場合のみ再計算
```

#### チェックのみ (再計算なし)

```
✅ 受注作成時: 税率チェック (ワーニング表示)
✅ 受注確定時: 税率チェック
```

### 8.7 消費税率の変更対応

#### 税率チェック対象

```
見積:
- 見積年月日の税率
- 締切日の税率
→ 両方が一致している必要がある

受注:
- 見積年月日の税率
- 締切日の税率
- 受注年月日の税率
→ すべてが一致している必要がある
```

#### エラー時の対応

```
保存時・申請時:
→ 編集可能 (その場で修正)

受注作成時・受注確定時:
→ 新バリエーション作成必須
```

#### 受注作成時のワーニング

```
シナリオ:
消費税率変更前の受注を
消費税率変更後に事後的に作成

対応:
ワーニング表示 + 確認ダイアログ
「消費税率が見積時と異なります。続けますか?」
```

### 8.8 見積書の金額表示

```
■ 明細項目
- 品名
- 数量
- 単位
- 単価
- 掛率
- 金額 (単価×掛率)
- 明細値引金額
- 最終明細金額

■ 明細フッタ
- 明細金額小計
- 明細値引金額小計
- 最終明細金額小計

■ フッタ
- 全体値引金額
- 全体値引後金額
- 消費税 (税率 + 端数処理)
- 消費税額
- 最終合計金額

■ 得意先向けの追加表示
- 粗利
- 粗利率
```

## 9. 権限管理

### 9.1 権限マトリクス

| 操作     | 誰ができる?             |
| -------- | ----------------------- |
| 見積作成 | 誰でも可能              |
| 見積編集 | 作成者 + 同部署メンバー |
| 見積申請 | 作成者 + 同部署メンバー |
| 見積承認 | (保留 - 後で決定)       |
| 受注作成 | 作成者 + 同部署メンバー |
| 受注編集 | 作成者 + 同部署メンバー |
| 受注確定 | 作成者 + 同部署メンバー |

### 9.2 部署の考え方

```
前提:
- ユーザーは1つの部署に所属
- 見積には作成者の部署情報が紐づく
- 同部署メンバーは協力して作業できる
```

### 9.3 権限チェックのタイミング

```
✅ API レベル: ミドルウェアで権限チェック
✅ ユースケースレベル: PermissionService で権限チェック
✅ UI レベル: ボタンの表示制御
```

## 10. メモ機能

### 10.1 メモの種類

#### 見積バリエーションレベル

| メモ     | 用途         | 見積書印字    | 編集権限       |
| -------- | ------------ | ------------- | -------------- |
| 客先メモ | 客先折衝履歴 | ✅ 印字する   | 見積編集権限者 |
| 内部メモ | 社内用メモ   | ❌ 印字しない | 見積編集権限者 |

#### 見積明細レベル

| メモ     | 用途                     | 見積書印字    | 編集権限       |
| -------- | ------------------------ | ------------- | -------------- |

## 12. 補足事項

### 12.1 締切管理

```
締切超過時の動作:
✅ 見積操作: 制限なし (参考情報のみ)
✅ ダッシュボード: 締切1週間前から通知・表示

通知条件:
- 締切日が1週間以内
- ステータスが「作成中」または「申請中」
- 担当者に通知
```

### 12.2 データ整合性の保証

```
見積と受注の関係:
✅ 受注作成後、見積バリエーションは編集不可
✅ 見積削除は禁止 (またはソフトデリート)
✅ 受注は見積データを参照 (参照整合性を保持)

金額計算:
✅ 保存時・申請時に自動再計算
✅ 計算結果をDBに保存
✅ 消費税率変更時はエラーチェック
```

### 12.3 パフォーマンス最適化

```
インデックス:
✅ 見積番号
✅ ステータス
✅ 作成者・部署
✅ 得意先・納品先
✅ 年度・区分・連番

非正規化:
✅ 見積に departmentId を保持 (権限チェック高速化)
✅ 計算結果を保存 (再計算不要)
```

## 付録 A: 主要な業務ルール

### A.1 見積作成のルール

```
1. 見積番号は保存時に自動採番
2. 見積作成者の部署が自動設定される
3. 消費税率は作成時点の税率が自動設定される
4. 納品先見積は申請・受注作成不可
5. 事後見積は承認不要
```

### A.2 バリエーションのルール

```
1. 1見積につき申請できるバリエーションは1つのみ
2. 無効状態のバリエーションは編集・申請不可
3. 申請中・承認済みのバリエーションは無効化不可
4. バリエーション番号は連番で自動採番
```

### A.3 受注作成のルール

```
1. 受注番号は持たず、見積番号で管理
2. 承認済みバリエーションからのみ作成可能 (事後見積除く)
3. 1バリエーションから1受注のみ作成可能
4. 受注作成後、見積バリエーションは編集不可
5. 納期は受注確定時に必須
```

### A.4 金額計算のルール

```
1. 明細金額 = 数量 × 単価 × 掛率 - 明細値引
2. 消費税は小計後に一括計算
3. 消費税の端数処理は選択式
4. 保存時・申請時に自動再計算
5. 消費税率変更時はエラーチェック
```

### A.5 得意先・納品先のルール

```
1. 納品先見積を先に作成
2. 「得意先改訂」で得意先見積を作成
3. 申請・受注作成は得意先見積のみ可能
4. 納品先価格 - 得意先価格 = 粗利
5. 実際の納品は納品先へ行う
```

## 付録 B: エラーメッセージ一覧

### B.1 見積作成時

| エラー                 | メッセージ                         | 対応         |
| ---------------------- | ---------------------------------- | ------------ |
| 修理見積で機器未設定   | 修理見積には修理対象機器が必須です | 機器を選択   |
| 事後見積で実施日未設定 | 事後見積には修理実施日が必須です   | 実施日を入力 |
| 事後見積で理由未設定   | 事後見積には緊急対応理由が必須です | 理由を入力   |

### B.2 バリエーション申請時

| エラー         | メッセージ                               | 対応                         |
| -------------- | ---------------------------------------- | ---------------------------- |
| 無効状態で申請 | 無効状態のバリエーションは申請できません | 有効化してから申請           |
| 明細なしで申請 | 明細が空のため申請できません             | 明細を追加                   |
| 重複申請       | 既に申請中のバリエーションが存在します   | 他のバリエーションを取り下げ |

### B.3 受注作成時

| エラー                 | メッセージ                                   | 対応                 |
| ---------------------- | -------------------------------------------- | -------------------- |
| 納品先見積から受注作成 | 納品先向け見積から受注作成できません         | 得意先改訂を実施     |
| 未承認で受注作成       | 承認されていない見積から受注を作成できません | 見積を承認           |
| 税率不一致             | 消費税率が異なります                         | 新バリエーション作成 |

### B.4 金額計算時

| エラー                         | メッセージ                               | 対応           |
| ------------------------------ | ---------------------------------------- | -------------- |
| 見積年月日と締切日で税率不一致 | 見積年月日と締切日で消費税率が異なります | 日付を修正     |
| 設定税率が不正                 | 消費税率が変更されています               | 税率を修正     |
| 最終金額がマイナス             | 値引き後の金額がマイナスになります       | 値引き額を調整 |
