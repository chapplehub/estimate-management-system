# ADR-0001: 申請テーブルを見積・価格で共通化する

| 起票日 | 2026-03-30 |

> **注記（2026-06-17）**: 本 ADR は [ADR-0053](0053-separate-estimate-application-table-from-shared.md) により差替えられた。申請テーブルは見積・価格共通ではなく **見積専用** に分離する。承認ロジックの共通化はテーブルではなくドメイン層で行う。経緯は ADR-0053 を参照。

## コンテキスト

承認フローは「見積」と「価格」の2種類に対して存在する。申請テーブルの設計として、種別ごとに別テーブルを作るか、共通の1テーブルにするかを決める必要がある。

別テーブルにすると外部キーが明確になるが、承認フローのロジック（承認ステップの管理・ステータス遷移・差戻処理など）が重複する。共通テーブルにすると承認ロジックを一元化できるが、対象IDがポリモーフィック（見積番号にも価格申請グループIDにもなる）になり、データベースレベルの外部キー制約を張れない。

## 決定

申請テーブルは見積・価格で共通の1テーブルとする。申請種別カラム（見積 / 価格）で区別し、対象IDの参照先は申請種別によって変わる。

## 影響

- 承認フローのロジック（申請→承認ステップ進行→差戻→取下）を共通化でき、コードの重複を避けられる
- 対象IDに対してデータベースレベルの外部キー制約を張れないため、アプリケーション層でのバリデーションが必要になる
- 将来的に新しい申請種別（例：受注確定の承認）を追加する場合、テーブルの変更なく申請種別の追加のみで対応できる
