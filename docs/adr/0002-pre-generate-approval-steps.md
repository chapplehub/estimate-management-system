# ADR-0002: 承認ステップを申請時に全ステップ事前生成する

| 項目 | 値 |
|---|---|
| ステータス | 採用 |
| 起票日 | 2026-03-30 |
| 最終更新日 | 2026-03-30 |

## コンテキスト

承認フローは複数ステップ（例：課長→部長→本部長）で構成される。承認ステップの生成方式として以下の2案がある。

1. **事前生成方式**: 申請時に承認チェーン全体を計算し、全ステップを一括で生成する
2. **逐次生成方式**: 前のステップが承認されてから次のステップを動的に生成する

逐次生成方式は承認チェーンの途中変更に柔軟だが、フロー全体の事前確認ができず、差戻時の履歴追跡が複雑になる。

## 決定

申請時に承認チェーン全体を計算し、全ステップを事前生成する。最初のステップのみ「承認待ち」、残りは「未着手」ステータスで生成する。

## 影響

- 申請時点で承認フロー全体を可視化でき、申請者や管理者がフローの全体像を確認できる
- 差戻時の履歴追跡が容易になる（全ステップが事前に存在するため）
- 申請時にチェーン上の全役割にユーザが割り当てられているかを検証でき、承認者不在によるフロー停滞を未然に防げる
- 承認チェーンの途中で組織変更（役割の上位役割変更など）があった場合、進行中の承認フローには影響しない（事前生成済みのため）
- 差戻後の再申請は新しい承認フローを最初から作り直すため、組織変更後の再申請では新しいチェーンが反映される
