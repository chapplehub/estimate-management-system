# ADR-0003: 高位役職者の申請でも常に上位役割に承認を求める

| 項目 | 値 |
|---|---|
| ステータス | 採用 |
| 起票日 | 2026-03-30 |
| 最終更新日 | 2026-03-30 |

## コンテキスト

承認フローのゴール（最終承認者の役職）は見積金額で決まる。例えば100万円以上は「部長承認」がゴールとなる。

ここで「部長自身が100万円の見積を申請した場合」に矛盾が生じる。申請者の上位役割は本部長レベルであり、ゴール（部長）よりスタート（本部長）が上位になる。

設計方針として以下の選択肢がある。

1. 申請者の役職 >= ゴール役職なら承認不要とする
2. 常に上位役割に承認を求める（ゴールとの比較に関係なく）
3. スタート > ゴールなら即承認（自動承認）

## 決定

承認者の役職レベルに関係なく、常にユーザテーブルの上位役割に対して承認を求める。「自分自身の見積は自分で承認できない」という暗黙のポリシーではなく、「常に上位に判断を仰ぐ」というポリシーを採用する。

ただし、承認ステップの役割に申請者自身が含まれている場合（兼務など）は自己承認を許可する。

## 影響

- 承認フローのロジックがシンプルになる（金額によるゴール判定と申請者の役職を比較する分岐が不要）
- 全ての申請に対して最低1ステップの承認が保証され、内部統制の観点で安全
- 部長が100万円の見積を作成した場合でも本部長の承認が必要になり、ゴールの金額レベルより厳しい承認になるケースがある
- 兼務ケース（例：部長と本部長の役割を持つユーザ）では自己承認が可能なため、実務上の柔軟性は確保される
