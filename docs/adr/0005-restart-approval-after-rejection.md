# ADR-0005: 差戻後の再申請は承認フローを最初からやり直す

| 項目 | 値 |
|---|---|
| ステータス | 採用 |
| 起票日 | 2026-03-30 |
| 最終更新日 | 2026-03-30 |

## コンテキスト

承認フローで差戻された見積を修正して再申請する場合、承認フローの扱いについて2つの方式がある。

1. **最初からやり直し**: 新しい承認フローを作成し、全ステップを再度通す
2. **途中再開**: 差戻されたステップから再開し、それ以前の承認は有効のまま

途中再開方式は承認者の負担を軽減するが、差戻後に見積金額が変更された場合にゴール（最終承認者の役職）が変わる可能性があり、既存ステップとの整合性が崩れる。

## 決定

差戻後の再申請は承認フローを最初からやり直す。新しい申請レコードと新しい承認ステップが生成される。

差戻された申請レコードと承認ステップは削除せず履歴として保持する。申請テーブルに「申請回数」（1, 2, 3...）を持たせ、同一対象に対する申請の履歴を `(対象ID, 申請種別, 申請回数)` で追跡可能にする。これにより承認者・申請者の双方が過去の差戻理由やコメントを確認できる。

## 影響

- 金額変更によるゴール変更を自然に処理できる（再申請時に再計算されるため）
- 組織変更（役割の上位役割変更）があった場合も、再申請時に最新の組織構造が反映される
- 承認者は再度全ステップで承認する必要があり、差戻と再申請が頻繁な場合は承認者の負担が増える
- 申請回数と対象IDにより、同一対象の申請履歴（過去の差戻理由・承認コメント含む）を時系列で参照できる
- 申請回数はシンプルな通番であり、表示（「第2回申請」）やクエリ（最新申請の取得）が容易
