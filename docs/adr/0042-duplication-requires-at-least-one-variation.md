# ADR-0042: 見積複製(C6)は最低1バリエーションの選択を要求する（§5.2「選択なし可」と §C1「空見積不可」の調停）

| 項目 | 値 |
|---|---|
| ステータス | 採用 |
| 起票日 | 2026-06-11 |
| 最終更新日 | 2026-06-11 |

## コンテキスト

システム設計書(見積) §5.2 は複製時のバリエーション選択について「**選択なしもOK（見積のみ複製）**」を許容している。一方、集約 Estimate は**空見積不可**を構造的不変条件として持つ（`Estimate.create` が variation 0 件を弾き、`removeVariation` も最後の 1 件を拒否、§C1）。両者は正面衝突する。

C6 DuplicateEstimate は「採番・保存をオーケストレーションする」**保存パス**であり（issue #295、§2.3 保存時採番）、空選択を許せば空 Estimate を保存しようとして `create()` が例外を投げる。

## 検討した選択肢

### A. C6 は最低 1 バリオーションの選択を要求する（採用）

選択 variation が 0 件なら `BusinessRuleViolationError` で拒否する。「見積のみ複製」は将来プレゼン層の「ヘッダ引き継ぎ新規作成」UX として実現し、保存責務の C6 からは分離する。

### B. 複製では空 Estimate を許可し不変条件を緩める（不採用）

§5.2 に忠実だが、空見積不可は全パス（作成・更新・削除）で効く核心不変条件であり、複製のためだけに緩めると集約の保証が壊れる。

### C. C6 を保存しないプレビュー DTO 生成に変更する（不採用）

空も表現できるが、issue の「採番・保存をオーケストレーション」と矛盾し、採番タイミング（§2.3 保存時採番）も変わる。

## 決定

A を採用する。C6 は保存パスとして最低 1 バリエーションの選択を要求し、0 件は業務ルール違反として拒否する。

## 根拠

空見積不可は集約の構造的不変条件であり、保存パスである C6 がこれを破る入力を受け付けるべきではない。§5.2 の「見積のみ複製」は、ヘッダ（得意先・提出区分・税率・メモ等）を引き継いだ新規作成というプレゼン層の UX として将来別途実現でき、保存責務から切り離せる。

## 影響

- §5.2 の「選択なしもOK」は C6（ドメイン・アプリ層）の責務外となる。設計書 §5.2 にその旨を注記する。
- 「見積のみ複製」UX は将来のプレゼン層 issue でヘッダ引き継ぎ新規作成として扱う。
