## 2. デザイン要約（v0.3.1）

### 2.1 目的・対象

1 件の見積を**閲覧し、その場で編集する**詳細画面。1 見積は複数の**バリエーション**（最大 99）を持ち、得意先との交渉過程で複数案を並行管理する。社内業務向けで情報密度高め、shadcn/ui + Tailwind、白基調＋青アクセント。

### 2.2 画面構造（上から下へ）

| # | 領域 | 主な中身 | 対応ドメイン |
|---|---|---|---|
| ① | 共通ヘッダーバー | ロゴ／ユーザ／サインアウト | （横断） |
| ② | タイトル行 | 戻るリンク・「見積詳細」・区分バッジ・承認状態バッジ（**プレースホルダ**）・見積番号・編集/保存/キャンセル・複製 | `Estimate` |
| ③ | 見積ヘッダーカード（タブ外） | 基本情報（得意先/納品先/日付/税率 など）＋ 修理情報（区分が修理系のときだけ折りたたみ表示） | `Estimate` / `RepairEstimateDetail` / `AfterRepairEstimateDetail` |
| ④ | バリエーションタブ | 連番タブ・アクティブ強調・無効はグレーアウト＋取消線・`+ 新規追加` | `EstimateVariation` |
| ⑤ | バリエーション操作行 | 複製・改訂・削除（無効化）・状態インジケータ・系譜表示 | `EstimateVariation` / 系譜 VO |
| ⑥ | 明細テーブル | 1行表示・横スクロール・商品名左sticky/最終金額右sticky・行アクティブ化・セット親子行・D&D | `EstimateItem` |
| ⑦ | 全体値引フィールド | `overallDiscount` 1項目 | `EstimateVariation` |
| ⑧ | メモエリア | 得意先向け/社内向けトグル | `EstimateVariation` |
| ⑨ | 金額サマリー（sticky フッター） | 明細小計/値引小計/最終小計/消費税/**最終合計**（アクティブバリエーションの集計） | `EstimateVariation` 集計値 |

> ③〜⑧は「タブ外（見積全体）」と「タブ内（選択中バリエーション）」に分かれる。②③が `Estimate`、④〜⑨が `EstimateVariation` 単位という二層構造が画面の骨格。

### 2.3 表示モード

- **閲覧モード（既定）**: 全フィールド読み取り専用、編集ボタン活性。
- **編集モード**: 編集ボタンで切替。フィールド活性化、保存／キャンセル表示。明細の追加・削除、バリエーション操作（複製/改訂/削除）は**編集モード中のみ**。
- 保存はクライアントで楽観計算 → サーバで確定計算（設計指示書 §10.3）。

### 2.4 行コンポーネント改良（v0.3.1 の主変更点）

本体指示書からの差分。明細テーブルの世界観が大きく変わっている。

| # | 変更 | 概要 |
|---|---|---|
| 1 | **行展開を廃止** | chevron トグル削除。基本金額・掛率適用後・得意先メモ・社内メモを**すべて列に昇格**し1行表示。テーブルは `overflow-x-auto`、商品名列を**左sticky**・最終金額列を**右sticky**。 |
| 2 | **行アクティブ化** | 行クリックで選択（薄青背景＋左青バー）。同時に1行のみ。タブ切替でリセット。 |
| 3 | **明細追加位置** | アクティブ行の直下に挿入（無ければ末尾）。追加後その行を自動アクティブ化。 |
| 4 | **セット親子行** | セット商品（`category === 'SET'`）選択でその行が**セット親行**化。配下に構成商品の**セット子行**を持つ。親は単価0/数量1固定/最終金額=子合計。子はインデント表示。`EstimateItem` に `kind`（normal/set-parent/set-child）と `parentItemId` を追加する前提。 |
| 5 | **D&D 強化** | 通常行は自由並替。セット親はグループごと移動。セット子は**同じ親の範囲内のみ**並替可（外に出せない・外から入れない）。 |

### 2.5 デザインが扱うデータ構造（指示書 §9 抜粋）

```
Estimate { estimateNumber, estimateType, estimateDate, deadline,
           submissionType,          ← ※後述の差分ポイント

               unit, unitPrice, discountRate, itemDiscount,

               revisedDetail?{deliveryPrice},

```

> 実装の所在: `src/server/subdomains/estimate/application/commands/*.ts`（コマンド）＋ `domain/entities/Estimate.ts`（集約メソッド）。
> 状態は 2026-06-13 時点（C1〜C4・C6・C7 実装済み、C5 のみ申請テーブル依存で未実装）。

| C | コマンド | デザイン上の操作（領域） | アプリ層ファイル | 実装状況 | 整合メモ |

### 4.1 対応表から見える重要ポイント（差分の予告）

1. **⑤バリ操作行「複製」に対応コマンドが無い**
   デザインの⑤複製は「現在のバリエーションを**内容ごと複製**して同一見積内に新バリエーション作成」（集約内）。一方 C3 `AddVariation` は**空**バリエーション追加、C6 は**集約またぎ**（新見積）、C7 は**改訂専用**。→ 「集約内バリエーション複製コマンド（仮 C3'）」が**未カバー**。

2. **C5（無効化）が未実装なのにデザインは完成済み**
   ④タブのグレーアウト・取消線、⑤削除（無効化）、⑤状態インジケータ（●有効/○無効）、§11「全バリエーション無効の警告」までデザインにある。実装は申請テーブル待ちでブロック。

3. **提出区分（submissionType）の所在ズレ**
   デザイン §4.1 は提出区分を**見積ヘッダー（Estimate領域）**に配置。実装は #323 リファクタで **`EstimateVariation` 単位**へ移動済み（ADR-0045、`EstimateVariation.ts:65`）。→ デザインは旧構造。画面でも**バリエーション側**に置き直す必要。

4. **セット親子行（kind / parentItemId）が未モデル化**
   改良指示書 §4 のセット親子は `EstimateItem` に `kind`・`parentItemId` の追加を要求するが、既存 `EstimateItem`（`domain/entities/EstimateItem.ts`）には**両フィールドとも存在しない**。商品サブドメインの `ProductCategory.SET` / `SetProductComponent` 連携も含め、ドメイン拡張が必要な**大きめの未カバー**。

> 1〜4 の深掘り（ズレ・不足の全件洗い出し）と本実装方針・issue分割案は **分析②** で扱う。
