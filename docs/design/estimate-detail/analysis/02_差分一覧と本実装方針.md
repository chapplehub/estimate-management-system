## 3. 本実装の方針メモ（合意済み設計判断）

1. **read-before-write**: 読み取り側もプレゼン層もゼロのため、閲覧モードの見積詳細画面（Q1＋DTO＋表示）を最初の独立スライス＝トレーサーバレットにする。以降に編集→バリ操作→セット編集を積む。
2. **セット明細は最初から織り込む（ADR-0047 / Shape ③-a）**: `estimate_items` を価格付き末端行（通常明細＋構成明細）1 表のまま据え置き、`estimate_set_items`（セット群＝価格・並び順・金額を持たない薄い衛星）と `estimate_set_components`（所属交差表・行の存在で表現＝NULL ゼロ）を新設。セット群の金額・位置は構成明細から導出。区分駆動の不変条件はドメイン層。
   - 命名規律: 「画面の親子」をそのまま「テーブルの親子」にしない（表現／ドメイン／永続化の三層分離）。詳細は `learning/画面の親子をそのままテーブルの親子にしない.md`。
3. **提出区分はバリ単位・表示専用**（ADR-0045）。詳細画面では編集 UI を持たず、作成(C3)時のみ確定。
4. **集約内バリ複製は C3 プリフィル**で実現（新コマンド・新系譜・新表なし）。
5. **C5 は今実装**（ADR-0018 パターン）、申請ガードはアプリ層の空の拡張点。状態**表示**は閲覧スライスに含める。
6. **周辺商品は通常明細**＋追加時サジェスト UX。セット商品（集約）と区別する。
7. **改訂明細の納品先価格は薄字表示**のみ。粗利は帳票の関心。
8. プレゼン実装は既存（customers/products）の **shadcn/ui + Tailwind + RSC + Server Action + conform-to** パターンを踏襲。クエリ層は product の queries を参照パターンにする。

## 4. 本実装 issue の分割案（1 画面 / 1 スライス単位）

依存順の 7 スライス。各スライスは独立 PR を想定。

| # | スライス | 層 | 主なスコープ | 依存 |
|---|---|---|---|---|
| **S1** | セット明細 基盤（ADR-0047） | BE | `estimate_set_items`＋`estimate_set_components` 追加（`estimate_items` 不変）／ドメイン（セット群・構成明細・不変条件）／Mapper 再構築／diff-upsert 拡張（ADR-0032）／ドメイン・リポジトリテスト | — |
| **S2** | 閲覧画面（read・トレーサーバレット） | BE+FE | `GetEstimateDetail`(Q1)＋表示 DTO（通常/構成明細・セット群導出金額・改訂 deliveryPrice・status・提出区分）／**読み取り専用詳細画面**（全 9 領域・1行表示・左右 sticky・セット群表示・改訂薄字・提出区分バッジ・状態表示・全無効警告）／seed にセット・改訂を含める | S1 |
| **S3** | 見積ヘッダー編集（C2） | FE+app | 編集モード切替／基本情報（日付・締切・得意先・納品先・部署・税率・端数）の編集・保存／修理情報の編集／税率チェック | S2 |
| **S4** | バリ内容編集・通常明細（C4） | FE+app | 通常明細の追加・削除・インライン編集・D&D（通常行）／全体値引／メモ（得意先/社内）／**周辺商品サジェスト**（本体追加で通常行として下に提案）／行アクティブ化・直下挿入 | S2 |
| **S5** | セット明細の編集（C1/C4 のセット書き込み） | FE+app | セット商品選択→構成明細自動展開（`SetProductComponent`）／set-aware D&D（群移動・構成は親範囲内のみ）／親削除カスケード／無効構成商品の警告／空セット群禁止 | S1, S4 |
| **S6** | バリ操作（C3/C6/C7） | FE+app | 新規追加（提出区分選択）／複製プリフィル（C3）／見積複製（C6・バリ選択 UI・最低1件 ADR-0042・単価クリア）／得意先改訂（C7・確認ダイアログ） | S2, S4 |
| **S7** | 無効化/有効化（C5） | FE+app | `ActivateVariationCommand`/`DeactivateVariationCommand`（ADR-0018）＋トグル UI／申請ガードの空の拡張点／無効タブのグレーアウト・取消線 | S2 |

> スコープ外（本実装に含めない）: 申請・承認 UI（プレースホルダのみ）、バリエーション比較ビュー、受注確定・取消、PDF 出力、粗利/粗利率の画面表示。

### 着手順の考え方

- **S1 → S2** が背骨（基盤→read 一本通し）。S2 完了でデモ可能な閲覧画面が立つ。
- S3 / S4 / S7 は S2 の上に並行着手可能（互いに独立）。
- S5 は S4（明細編集の土台）と S1（セット基盤）に依存。
- S6 は既存コマンド（C3/C6/C7）の UI 配線が主。

## 5. 残課題（本実装スライス内で詰める細目）

- S5: 通常商品 → セット商品への切替時の既存値（メモ・数量）の扱い（行改良指示書 §8 残課題）。
- S5: 無効構成商品の警告 UI（インライン or トースト）。
- S4/S5: メモ全文編集のポップオーバー要否（暫定はインライン Textarea rows=2）。
- S6: C6 のバリ選択ダイアログの具体 UI（単価クリアの明示）。
- 申請テーブル（ADR-0001）実装時に、C5 のアプリ層ガードを接続する。
