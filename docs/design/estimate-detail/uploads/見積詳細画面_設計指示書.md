# 見積詳細画面 デザイン指示書

社内業務アプリケーション「見積管理システム (ESM)」の見積詳細画面のレイアウト設計指示書。Claude Design で UI を生成するための入力ドキュメント。

---

## 1. 画面の目的と全体像

### 1.1 目的

1 件の見積を閲覧し、必要に応じてその場で編集する。1 件の見積は複数の **バリエーション**（最大 99 件）を持ち、得意先との交渉過程で複数案を並行管理する。

### 1.2 表示モード

- **閲覧モード（デフォルト）**: 全フィールドが読み取り専用、編集ボタンが有効
- **編集モード**: 編集ボタンクリックで切替、全フィールドが入力可能、保存／キャンセルボタンが表示

明細の追加・削除、バリエーション操作も編集モード中のみ可能。

## 2. デザイントーン

連携いただいた得意先一覧／編集画面に揃える。

- ベース: shadcn/ui + Tailwind CSS
- 配色: 白基調、アクセントは青（編集・保存ボタン）、注意は赤（削除・サインアウト）
- 角丸: 中程度（`rounded-md`〜`rounded-lg`）
- 影: カード境界は薄い影 or 薄いボーダー
- フォント: 標準（システムフォント）、見出しは太字
- 情報密度: 高め（社内業務向け、余白を取りすぎない）
- バッジ: 小さめのピル形（緑＝有効、グレー＝無効など）

## 3. 上部タイトル行

### 3.1 構成要素（左から右へ）

| 要素 | 内容 | 表示条件 |
|---|---|---|
| 戻るリンク | `← 見積一覧に戻る` | 常時 |
| タイトル | `見積詳細` | 常時 |
| 区分バッジ | `新規` / `修理` / `事後修理` | 常時 |
| 承認状態バッジ | `承認状態: 未実装` | 常時（プレースホルダー） |
| 見積番号 | `見積番号: N2500001` | 常時 |
| 編集ボタン | `編集` / 編集中は `保存` `キャンセル` | 常時（編集中は切替） |
| 複製ボタン | 見積全体を複製して新規見積を作る | 閲覧モードのみ |

### 3.2 編集モード切替時の挙動

- `編集` 押下 → 全入力フィールドが activate、ボタンが `保存` + `キャンセル` に切替
- `キャンセル` → 確認ダイアログ「変更を破棄しますか？」（変更があるときのみ）
- `保存` → バリデーション後、PUT 送信、成功時に閲覧モードに戻る

## 4. 見積ヘッダー情報（タブ外・上部カード）

`Estimate` モデルのフィールドを表示。1 つのカード内で 2 セクションに分かれる。

### 4.1 基本情報セクション

2 カラムのフォームグリッド（得意先編集画面と同じレイアウト）。

| ラベル | フィールド | 編集可否 | 備考 |
|---|---|---|---|
| 見積番号 | `estimateNumber` | 不可 | 自動採番、表示のみ |
| 見積区分 | `estimateType` | 不可 | 新規／修理／事後修理。作成時に確定、変更不可 |
| 提出区分 | `submissionType` | 編集可 | 得意先向け／納品先向け（ラジオ or セレクト） |
| 見積年月日 | `estimateDate` | 編集可 | 日付ピッカー |
| 締切年月日 | `deadline` | 編集可 | 日付ピッカー |
| 得意先 | `customer.company.name` + コード | 編集可 | コンボボックス（コードまたは名前で検索） |
| 納品先 | `deliveryLocation.company.name` + コード | 編集可 | 得意先選択後に絞り込み |
| 部署 | `department.name` | 編集可 | セレクト |
| 作成者 | `creator.name` | 不可 | 表示のみ |
| 消費税率 | `taxRate` (%) | 編集可 | スピナー or 入力 |
| 端数処理 | `taxRoundingType` | 編集可 | 切捨／切上／四捨五入のラジオ |

### 4.2 修理情報セクション（区分が REPAIR / AFTER_REPAIR のときのみ表示）

折りたたみカード（`Collapsible`）。デフォルト閉じる、ヘッダー部分に対象機器名と修理日の要約を表示。

#### REPAIR（事前修理）の場合

| ラベル | フィールド | 必須 |
|---|---|---|
| 修理対象機器 | `repairDetail.targetProduct` | ◯ |
| 故障内容 | `repairDetail.faultDescription` | ◯ |
| 修理予定日 | `repairDetail.scheduledRepairDate` | ◯ |

#### AFTER_REPAIR（事後修理）の場合

| ラベル | フィールド | 必須 |
|---|---|---|
| 修理対象機器 | `afterRepairDetail.targetProduct` | ◯ |
| 故障内容 | `afterRepairDetail.faultDescription` | ◯ |
| 修理実施日 | `afterRepairDetail.actualRepairDate` | ◯ |
| 緊急対応理由 | `afterRepairDetail.emergencyReason` | ◯ |
| 10 万円超警告確認済 | `afterRepairDetail.afterServiceWarningAcknowledged` | チェックボックス |

故障内容・緊急対応理由はそれぞれ最大 2000 文字。`Textarea` を使い、初期高さは 3〜4 行程度。

## 5. バリエーションタブ領域

### 5.1 タブの構成

- タブラベル: `バリエーション 1` のような連番表示（`variationNumber`）
- 無効化されたバリエーション（`status = INACTIVE`）はタブ自体をグレーアウト＋取消線、または非表示切替（フィルタトグルで「無効を表示」）
- アクティブタブには `*` または下線などで視認性を上げる
- タブの右端に `+ 新規追加` ボタン

### 5.2 バリエーション操作行

タブ直下に操作行を置く。

| 操作 | 動作 | 編集モード必須 |
|---|---|---|
| 複製 | 現在のバリエーションを複製して新規バリエーション作成 | あり |
| 改訂 | 得意先改訂を起票（このバリエーションを元に改訂版を作成） | あり |
| 削除（無効化） | `status` を `INACTIVE` に変更 | あり |

操作行の右側に **状態インジケータ**（`●有効` / `○無効`）と、複製・改訂で生まれたバリエーションの場合は **系譜表示**（例: `← バリエーション 2 から複製`）を表示する。

### 5.3 タブ内のレイアウト構成

1. バリエーション操作行
2. 明細テーブル
3. 明細追加ボタン（編集モードのみ）
4. 全体値引フィールド
5. メモエリア

## 6. 明細テーブル

### 6.1 列構成（主表示）

| 列名 | フィールド | 幅目安 | 編集可否 |
|---|---|---|---|
| `▼` | 行展開トグル | 32px | - |
| `#` | sortOrder | 48px | ドラッグ&ドロップで並び替え（編集時） |
| 商品名 | `itemName` | 広め | 商品検索コンボボックス（商品マスタから選択、選択時に名称・単価・単位を取得しスナップショット） |
| 数量 | `quantity` | 80px | 数値入力 |
| 単位 | `unit` | 60px | 表示のみ（商品選択時に決まる） |
| 単価 | `unitPrice` | 100px | 数値入力 |
| 掛率 | `discountRate` | 80px | 数値入力（1.0 = 値引なし、デフォルト 1.0） |
| 値引 | `itemDiscount` | 100px | 数値入力（明細値引金額） |
| 最終金額 | `finalAmount` | 120px | 表示のみ（自動計算） |
| `×` | 行削除 | 32px | 編集モードのみ |

### 6.2 行展開時の表示（折りたたみ詳細）

行の `▼` をクリックで展開し、以下を表示。

- **基本金額** (`baseAmount`): 数量 × 単価
- **掛率適用後金額** (`discountedAmount`): 基本金額 × 掛率
- **得意先向けメモ** (`customerMemo`): Textarea、最大 2000 文字
- **社内向けメモ** (`internalMemo`): Textarea、最大 2000 文字

行展開は閲覧・編集どちらのモードでも可能。

### 6.3 金額表示フォーマット

- 通貨記号: `¥`
- 3 桁区切りカンマ: `¥1,234,567`
- 円未満切捨で表示（内部は銭単位の整数だが、画面では円単位）
- 負の金額は赤字（値引列で使う場合）

### 6.4 改訂明細の特別表示

`EstimateItem.revisedDetail` が存在する明細（得意先改訂で生まれた明細）は、行の左端に小さなマーカー（例: 紫の縦線）を表示。展開時に「納品先価格: ¥XXX」を追加表示。

## 7. 全体値引・メモエリア

### 7.1 全体値引

`EstimateVariation.overallDiscount` を編集する 1 フィールド。明細テーブルの直下に配置。

```
全体値引: [          ¥0 ]  （編集モード時は入力可）
```

### 7.2 メモエリア

`EstimateVariation.customerMemo` と `internalMemo` をトグル切替で表示。

```
[ 得意先向け* ] [ 社内向け ]   ← トグルボタン群
┌──────────────────────────────────────┐
│ （選択された方のメモ本文を表示）       │
│                                       │
└──────────────────────────────────────┘
```

それぞれ最大 2000 文字、Textarea で 4〜5 行の初期高さ。

## 8. 金額サマリー（sticky フッター）

画面下部に常時固定表示。スクロールしても消えない。

### 8.1 表示項目

`EstimateVariation` の集計フィールドを左から右へ：

| ラベル | フィールド | 強調 |
|---|---|---|
| 明細小計 | `subtotal` | 通常 |
| 値引小計 | `discountSubtotal` | 通常 |
| 最終小計 | `finalSubtotal` | 通常 |
| 消費税 | `taxAmount` | 通常 |
| **最終合計** | `finalTotal` | **太字＋大きめ** |

### 8.2 レイアウト

横一列、`flex justify-between` or `grid`。右端に最終合計を一番大きく配置。

```
┌─────────────────────────────────────────────────────────────┐
│ 明細小計 ¥X,XXX,XXX  値引小計 ¥XX,XXX  最終小計 ¥X,XXX,XXX │
│ 消費税 ¥XXX,XXX                  最終合計 ¥X,XXX,XXX       │
└─────────────────────────────────────────────────────────────┘
```

2 行構成にしてもよい。最終合計だけ別行で大きく見せる案も可。

### 8.3 アクティブタブの金額を表示

サマリーは現在アクティブなバリエーションの金額。タブ切替で更新される。

## 9. データ構造（schema からの抜粋）

画面で扱う型のサマリー。フル schema は別ファイル `schema.prisma` を参照。

```ts
// 見積（タブ外）
Estimate {
  id, estimateNumber, estimateType: NEW|REPAIR|AFTER_REPAIR,
  estimateDate, deadline,
  submissionType: CUSTOMER|DELIVERY_LOCATION,
  customer: { code, name },
  deliveryLocation: { code, name },
  department: { name },
  creator: { name },
  taxRate, taxRoundingType: ROUND_DOWN|ROUND_UP|ROUND,
  repairDetail?: { ... },         // REPAIR のときのみ
  afterRepairDetail?: { ... },    // AFTER_REPAIR のときのみ
  variations: EstimateVariation[],
}

// バリエーション（タブ内）
EstimateVariation {
  id, variationNumber, status: ACTIVE|INACTIVE,
  customerMemo, internalMemo,
  overallDiscount,
  subtotal, discountSubtotal, finalSubtotal, taxAmount, finalTotal,
  copyTarget?, revisionTarget?,   // 系譜情報
  items: EstimateItem[],
}

// 明細
EstimateItem {
  id, sortOrder,
  product: { code, name },
  itemName, quantity, unit, unitPrice,    // 商品スナップショット
  customerMemo, internalMemo,
  discountRate, itemDiscount,
  baseAmount, discountedAmount, finalAmount,
  revisedDetail?: { deliveryPrice },      // 改訂明細のみ
}
```

## 10. インタラクション仕様

### 10.1 ボタン階層

| 重要度 | 用途 | スタイル |
|---|---|---|
| Primary | 保存、編集開始 | 青背景・白文字 |
| Secondary | キャンセル、複製、改訂 | グレー枠・グレー文字 |
| Destructive | 削除（無効化） | 赤背景・白文字 |
| Ghost | 戻るリンク、行展開、タブ操作 | 文字のみ |

### 10.2 確認ダイアログが必要な操作

- 編集キャンセル（変更ありの場合）
- バリエーション削除（無効化）
- 改訂起票

### 10.3 自動計算

編集モードで以下の値を変更したとき、関連する金額を即時再計算して表示：

- `quantity`, `unitPrice` 変更 → `baseAmount` → `discountedAmount` → `finalAmount` → サマリー
- `discountRate`, `itemDiscount` 変更 → `discountedAmount` → `finalAmount` → サマリー
- `overallDiscount` 変更 → サマリー
- 税率変更 → `taxAmount` → `finalTotal`

実装は client 側で楽観的に計算し、保存時にサーバ側で確定計算する。

## 11. 状態遷移と表示パターン

| 場面 | 表示 |
|---|---|
| 編集モード | フィールド枠が活性、保存/キャンセルボタン表示 |
| 閲覧モード | フィールドが読み取り専用（背景グレー or 枠なし） |
| 無効バリエーション | タブがグレーアウト＋取消線、内容は読み取り専用 |
| 全バリエーション無効 | 警告メッセージ「すべて無効化されています」 |
| 修理系見積 | 修理情報カードがヘッダーに追加表示 |
| 改訂明細 | 行の左に紫マーカー、展開時に納品先価格表示 |

## 12. 想定しないもの（スコープ外）

- 申請・承認 UI の本実装（プレースホルダーのみ）
- バリエーション間の比較ビュー
- 受注確定・取消の操作
- PDF 出力（ボタンだけ将来用に置くなら可）
