## 1. 行展開廃止と 1 行表示

### 1.1 列構成（左→右）

行展開は廃止し、すべての項目を 1 行に並べる。テーブル全体は横スクロール可能にする。

| 列名 | 旧扱い | 新扱い | 幅目安 |
|---|---|---|---|
| 行マーカー（改訂・状態） | 列 | 列（維持） | 8px |
| ドラッグハンドル | 列（編集時のみ） | 列（維持） | 24px |
| # | 列 | 列（維持） | 40px |
| 商品名 | 列 | **左 sticky**（横スクロール時固定） | 220px〜 |
| 数量 | 列 | 列 | 80px |
| 単位 | 列 | 列 | 56px |
| 単価 | 列 | 列 | 112px |
| 掛率 | 列 | 列 | 80px |
| 値引 | 列 | 列 | 112px |
| **基本金額** | 行展開内 | **列に昇格** | 128px |
| **掛率適用後** | 行展開内 | **列に昇格** | 128px |
| 最終金額 | 列 | **右 sticky**（横スクロール時固定） | 128px |
| **得意先向けメモ** | 行展開内 | **列に昇格** | 200px |
| **社内向けメモ** | 行展開内 | **列に昇格** | 200px |
| 削除ボタン | 列（編集時のみ） | 列（維持） | 32px |

合計幅が広いため、テーブル本体は `overflow-x-auto` のラッパー内に置く。
**商品名列を左 sticky**、**最終金額列を右 sticky** にすることで、横スクロール中も主要情報が見える。

### 1.2 メモ列の表示形式

- **閲覧モード**: 1〜2 行に省略表示（`line-clamp-2`）。ホバーで tooltip 全文 or クリックでポップオーバー展開。
- **編集モード**: 列幅内に収まる `<Textarea>`（rows=2）でインライン編集。`max-h` をつけて行高を抑える。

メモ全文を編集したい場合に備え、行右端などに「メモを別画面で編集」ボタンを置く案もあるが、まずはインライン編集で十分。

### 1.3 行展開トグル（chevron）

廃止。`▶ / ▼` ボタンは削除する。改訂マーカー（紫の縦線）は維持。

## 2. 行のアクティブ化（選択状態）

### 2.1 動作

- 行（`<tr>`）をクリックすると、その行がアクティブ状態になる。
- アクティブ行は **同時に 1 行だけ**。別の行をクリックすると切り替わる。
- 編集モード／閲覧モード問わずアクティブ化できる。
- 入力フィールド（input/textarea/button）をクリックしてもアクティブ化は発火する。ただしフォーカスは入力に取られて良い。

### 2.2 解除条件

- 別の行をクリック → その行に切り替わる
- 同じ行をもう一度クリック → 解除はしない（アクティブ維持）
- テーブル外をクリック → **解除しない**（明細追加ボタンの挙動が壊れるため）
- バリエーションタブを切り替え → 解除（タブごとにアクティブ行は独立）

### 2.3 ビジュアル

- 背景色を `bg-blue-50` 程度に変更
- 左端に細いインジケータバー（`border-l-2 border-blue-500`）
- ホバーとの違いを明確に（ホバーは薄い neutral、アクティブは青系）

セット親行・セット子行も同様にアクティブ化可能。アクティブの粒度は **1 行単位**（親をクリックしても子はアクティブにならない）。

## 3. 明細追加ボタンの挙動

### 3.1 挙動マトリクス

| アクティブ行の状態 | 追加位置 |
|---|---|
| アクティブ行なし | テーブル末尾（現状通り） |
| 通常行がアクティブ | その行の **直下** |
| セット親行がアクティブ | セット親グループの **直下**（最後の子行のさらに下、つまりセット範囲の外） |
| セット子行がアクティブ | **そのセットの子行として** 直下に追加 |

セット子行をアクティブにして「明細追加」を押すと、新規行も同じ親の子行として追加される。これによりセット内に手動で構成商品を追加できる（設計書 §4.3 の自動追加の補完操作）。

### 3.2 追加される行の初期状態

- 種別: 「通常行」または「セット子行」（上記マトリクスに従う）
- 商品: 未選択
- 数量・単価・掛率・値引: デフォルト値（数量1、掛率1.0、値引0）
- 追加直後、その行は **自動的にアクティブになる**（連続して追加するときに便利）

### 3.3 追加後の sortOrder 振り直し

挿入位置に応じて `sortOrder` を 1 から振り直す。セット親子のグループ順は崩さない。

## 4. セット商品用の親子行コンポーネント

### 4.1 概要

商品設計書 §4 のセット商品（`ProductCategory.SET`）が選択された明細行は、通常行から **セット親行** に切り替わる。セット親行は配下に **セット子行**（構成商品）を持つ。

### 4.2 データモデル拡張

`EstimateItem` に種別と親参照を追加する。

```ts
type EstimateItemKind = 'normal' | 'set-parent' | 'set-child';

interface EstimateItem {
  id: string;
  sortOrder: number;
  kind: EstimateItemKind;           // 追加
  parentItemId: string | null;       // 追加（set-child のときのみ親 id）
  // 既存フィールド
  product: { id, code, name, category };
  itemName, quantity, unit, unitPrice,
  discountRate, itemDiscount,
  customerMemo, internalMemo,
  ...
}
```

実装上はフラット配列で持ち、`parentItemId` で親子を表現する。レンダリング時にグルーピングする。

セット親行の `unitPrice` は **常に 0**（商品設計書 §2.2「セット商品: 価格の設定不可、常に 0 円」）。
セット親行の `quantity` は **常に 1 固定、編集不可**。UI 上は数量列に `1` を表示するが、入力フィールドではなく読み取り専用の表示にする。
セット親行の **最終金額は子行の合計**。

### 4.3 商品選択時の自動切替

明細の商品選択コンボボックスでセット商品（`category === 'SET'`）を選んだ場合：

1. その行の `kind` を `'set-parent'` に変更
2. `unitPrice` を 0 に、`discountRate` を 1.0 に、`itemDiscount` を 0 に強制
3. 商品マスタの `setComponents`（`SetProductComponent[]`）を読み、各構成商品を子行として親の直下に挿入
4. 各子行の数量は `SetProductComponent.quantity` の値（商品設計書 §4.3）
5. 構成商品に **無効商品が含まれる場合は警告を表示**（商品設計書 §5.2「無効な構成商品も追加される、警告表示」）

通常商品 → セット商品への切替時、既存の値（メモなど）は破棄して良いか？ ここは要確認。暫定で「商品変更時は通常の挙動どおりスナップショットを更新、メモは保持」とする。

### 4.4 セット親行の表示

- 行頭にセットアイコン（例: `<Boxes>` や `<Package>` 系）と「セット」ラベル
- 背景色: ほんのり別色（例: `bg-amber-50/30`）でグループの始まりを示す
- 子行を **折りたたむトグル** を行頭に置く（▼で展開、▶で折りたたみ）。デフォルトは展開。
- 数量・単価・掛率・値引・基本金額・掛率適用後 → **すべて非活性 or 非表示**
- 最終金額列 → 子行の合計を表示（太字）

### 4.5 セット子行の表示

- 商品名列を **インデント**（左パディング +24px 程度、左に細い縦線で親との所属を視覚化）
- 行頭に小さな `└` 記号や、親のセットアイコンの簡略版
- `kind === 'set-child'` のときも、数量・単価・掛率・値引・メモはすべて編集可能（普通の明細と同じ振る舞い）
- 削除ボタンを押すと、その子行のみが削除される（親は残る）

### 4.6 セット親行の削除

セット親行の削除ボタンを押した場合、**子行ごとまとめて削除** する。
編集モードのとき、削除確認ダイアログを出す案もある（破壊的操作）。暫定では削除確認なし、編集モードのキャンセルで元に戻せる前提。

### 4.7 子行の折りたたみ状態

- バリエーションタブ切替で **リセット**（毎回展開状態に戻る）
- アクティブ行は折りたたみ状態と独立（折りたたんでもアクティブは維持）

## 5. D&D ルール

### 5.1 並び替え可能性マトリクス

| ドラッグ元 → ドロップ先 | 通常行の位置 | セット親行の位置 | セット子行の位置（同じ親） | セット子行の位置（別の親） |
|---|---|---|---|---|
| 通常行 | ✅ 可 | ✅ 可（セット親の前後） | ❌ 不可 | ❌ 不可 |
| セット親行 | ✅ 可（子行を引き連れて） | ✅ 可（子行を引き連れて） | ❌ 不可 | ❌ 不可 |
| セット子行 | ❌ 不可（外に出せない） | ❌ 不可（別グループに移動不可） | ✅ 可（親範囲内のみ） | ❌ 不可 |

### 5.2 セット親行のドラッグ

- セット親行をドラッグすると、**配下の子行もまとめて移動** する（グループ移動）
- ドラッグ中のゴーストには「セット商品X（構成 N 件）」のような表示があると親切

### 5.3 セット子行のドラッグ

- 子行のドラッグハンドルは編集モードのみ表示
- ドロップ可能な位置は **同じ親の子行の範囲内のみ**
- 範囲外の位置にホバーしているとき、視覚的に「ここには落とせない」ことを示す（カーソル `not-allowed`、ドロップインジケータ非表示）

### 5.4 ドラッグ視覚フィードバック

- ドラッグ中の元行: 半透明（`opacity-50`）
- ドロップ可能な位置: 行間に青いインジケータバー
- ドロップ不可能な位置: 青いインジケータを出さない、カーソル `not-allowed`

### 5.5 実装ヒント

フラット配列上で D&D する場合、ドロップ判定は以下のロジック：

```
ドロップ可否(from, to):
  if from.kind === 'set-child':
    return to.parentItemId === from.parentItemId
  if from.kind === 'set-parent' or 'normal':
    return to.kind !== 'set-child'   # 子行の途中には落とせない
```

並び替え後は、グループの整合性を保つため：
- セット親が動いたら子行も追従させて挿入
- 子行の移動は、同じ親内での位置交換のみ

## 6. 既存仕様への影響まとめ

| 既存仕様 | 変更 |
|---|---|
| 行展開トグル（chevron） | 廃止 |
| 行展開エリアの表示内容 | 全部 1 行に統合（基本金額・掛率適用後・メモ × 2） |
| メモのトグル表示（得意先向け/社内向け） | バリエーションのメモは現状維持。明細のメモは 1 行表示なので両方並列。 |
| 改訂明細マーカー（紫線） | 維持 |
| 改訂明細の納品先価格 | 行内のどこに出す？ → メモ列の隣などに `納品先価格` 列を **改訂明細があるバリエーションのときだけ追加** する案、または最終金額列の下にツールチップで補足。暫定: 改訂明細がある明細だけ最終金額の下に薄く `納品先 ¥XXX` を表示。 |
| 金額サマリー（sticky フッター） | セット親行は単価0、子行が金額を持つ。サマリー集計は **すべての行（親含むが親は0）の合計** で結果同じ。実装上はそのままで OK。 |

## 7. Claude Design への依頼文（コピペ用）

> 前回プロトタイプ「見積詳細画面」の明細テーブル（items-table.jsx）を改良します。
> 添付の差分指示書に従い、以下を実装してください。
>
> 1. 行展開（chevron トグル）を廃止し、基本金額・掛率適用後・得意先メモ・社内メモを **すべて 1 行内の列として** 表示。テーブルは横スクロール可能にし、商品名列を左 sticky、最終金額列を右 sticky にする。
> 2. 行クリックでその行をアクティブ化（薄い青背景＋左の青いバー）。アクティブは同時に 1 行のみ。
> 3. 「明細追加」ボタンは **アクティブ行の直下** に挿入。アクティブ行がなければ末尾。セット子行がアクティブなら同じセットの子行として挿入。
> 4. セット商品（`product.category === 'SET'`）が選択された行は **セット親行** になり、配下に **セット子行**（構成商品）を持つ。親は単価0、最終金額は子の合計。子行はインデント表示。
> 5. D&D ルール: 通常行は自由に並べ替え可。セット親はグループごと移動。セット子は **同じ親の子行範囲内でのみ** 並べ替え可、外には出せず外からも入れない。
>
> ダミーデータにセット商品の明細を 1〜2 件含めて、見え方を確認できるようにしてください。
