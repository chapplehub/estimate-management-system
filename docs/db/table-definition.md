# テーブル構成

## テーブル構成の凡例

R は必須行
P はプライマリキー
U はユニークキー
CU は複合ユニークキー
F は外部キー

### 見積

- 見積 ID(P)
- 元見積 ID(F)
- 見積ステータス ID(R,F)
- 見積区分 ID(R,F)
- 見積年月日(R)
- 締切年月日(R)
- 取引先区分 ID(R)　(画面では提示先区分)
- 得意先取引先 ID(F)
- 納品先取引先 ID(F)

### 見積明細

- 見積明細 ID(P)
- 商品 ID(R,CU)
- 数量(R)
- 見積書出力区分(R)

#### 見積ステータス

- 進行中
- 申請中
- 差戻
- 取下
- 受注済

#### 取引先区分

- 得意先
- 納品先

### 受注(見積と統合するか考え中)

- 見積 ID(P)
- 受注ステータス ID(R,F)
- 見積区分 ID(R,F)
- 見積年月日(R)
- 締切年月日(R)
- 提示先区分 ID(R)
- 得意先取引先 ID(F)
- 納品先取引先 ID(F)

### 受注明細

- 受注明細 ID(P)
- 見積明細 ID(F)
- 商品 ID(R,CU)
- 数量(R)
- 見積書出力区分(R)
- 受注ステータス ID(R,F)

#### 受注ステータス

- 進行中
- 確定済
- 変更禁止
- 取下

#### 価格申請グループ(取引先価格の申請単位に名前を付ける)

- 価格申請グループ ID(R,P)
- 申請理由区分(R)：
- 申請理由詳細(R)
- 価格申請グループ名(R)

#### 申請理由区分

- 他社競合による値下げ
- 原材料高騰による値上げ

##### 価格

- 価格 ID(P)
- 商品 ID(R,CU)
- 年月日(R,CU)

##### 標準価格

- 価格 ID(P,F)
- 標準価格(R)

##### 取引先価格

- 価格 ID(P,F)
- 価格申請グループ ID(R)
- 取引先区分 ID(R,CU)
- 取引先 ID(R,CU)
- 取引先価格(R)

### 取引先区分

- 得意先
- 納品先

### 申請(要検討)

- 申請 ID(P)
- 申請区分 ID(R,F)
- 申請ステータス区分
- 申請ユーザ ID(R,F)

### 承認(要検討)

- 承認 ID(P)
- 申請 ID(F)
- 承認ユーザ ID(R,F)

### 申請区分

- 見積
- 価格

### 商品

- 商品 ID(P)
- 商品区分 ID(F)
- 商品名(R)
- 単位 ID(R, F)

### 商品周辺商品

- 商品 ID(CU,R,F)
- 周辺商品 ID(CU,R,F)

### セット商品

- セット商品 ID(CU,R,F)
- 構成商品 ID(CU,R,F)

### 商品区分

- 個別商品
- 消耗品
- セット商品

#### ユーザ

- ユーザ ID(P)
- 上位役割 ID(F)
- ユーザ名(R)
- 部署 ID(F)

#### 部署

- 部署 ID(P)
- 部署 CD(U)
- 部署名

#### 役割

- 役割 ID(P)
- 上位役割 ID(F)
- 役職 ID(R)
- 役割名(R)：大阪市南課長・大阪市部長など

上位役割に指定できるのは役職の上位役職に含まれる役割のみという制限

#### 役職

- 役職 ID(P)
- 上位役職 ID(F)
- 役職名(P)：課長・部長など

#### 役職は現時点で以下の４つ

- 課長
  - 大阪市南課長など
  - 上位役職：部長
- 部長
  - 大阪市部長など
  - 上位役職：本部長
- 本部長
  - 関西本部長など
  - 上位役職：本部長
- 社長
  - 社長のは「社長」のみ
  - 一番上の役職
  - 上位役職：なし
