# Issue #240: 納品先一覧画面の得意先パターン準拠 — 実装計画

## 概要

PR #248 で得意先一覧に対して行われた UI 変更パターンを、納品先一覧にも適用する。

主な変更:
- h1「納品先管理」→「納品先一覧」（得意先パターン準拠）
- h2「納品先一覧」ラッパーの削除
- 「新規登録」ボタンの追加
- E2E テストの修正

スコープ外: 詳細画面の変更、新規作成・編集・削除画面（→ #241 に調査結果を追記済み）

## 設計判断

### 「新規登録」ボタンのリンク先
- `/delivery-locations/new` にリンクするが、作成ページは #241 で実装予定
- 得意先パターン踏襲（一覧ページの構造を先に合わせる）のため判断不要

## ステップ

### Step 1: 一覧ページの更新（h1変更 + h2削除 + 新規登録ボタン）
- 対象ファイル:
  - `src/app/(features)/delivery-locations/page.tsx` (修正)
- 作業内容:
  - h1「納品先管理」→「納品先一覧」
  - h1 の横に「新規登録」Link ボタンを追加（`/delivery-locations/new`、得意先一覧ページと同一スタイル）
  - h2「納品先一覧」とその wrapper `<div className="px-8 pt-6 pb-2">` を削除
  - DataTable の外側コンテナに `pt-4` を追加（得意先一覧ページと同一構造）
- コミットメッセージ: `feat: 納品先一覧ページのh1変更・h2削除・新規登録ボタンを追加`

### Step 2: E2E テストの修正
- 対象ファイル:
  - `src/app/(features)/delivery-locations/delivery-locations-list.e2e.ts` (修正)
- 作業内容:
  - `waitForListReady` ヘルパー関数（L9）: `"納品先管理"` → `"納品先一覧"`
  - テスト「一覧が表示される」（L24）: `"納品先管理"` → `"納品先一覧"`
  - テスト「一覧が表示される」（L25）: h2「納品先一覧」アサーションを削除（h2 が削除され、h1 が「納品先一覧」に変更されるため重複）
  - テスト「一覧が表示される」（L32）: `not.toBeVisible()` → `.toBeVisible()`（「新規登録」ボタンが追加されるため）
  - テスト「コードリンクから詳細画面に遷移できる」（L154）: 変更なし（詳細画面の h1 は未変更）
  - テスト「一般ユーザーでも納品先一覧を閲覧できる」（L174）: `"納品先管理"` → `"納品先一覧"`
- コミットメッセージ: `test: 納品先一覧E2Eテストをh1変更・新規登録ボタン追加に対応`

### Step 3: Lint チェック
- 作業内容:
  - `pnpm lint` 実行
  - エラーがあれば修正
- コミットメッセージ: (必要時) `fix: lint エラーの修正`

## 検証方法

1. `pnpm dev` で開発サーバー起動
2. ブラウザ確認:
   - `/delivery-locations` — h1「納品先一覧」に変更、h2 がなくなっている
   - 「新規登録」ボタンが表示され、`/delivery-locations/new` へリンクしている
   - テーブルレイアウトが得意先一覧（`/customers`）と同一構造
3. `pnpm e2e -- delivery-locations-list` で E2E テストがパス
4. `pnpm lint` パス
