# Issue #240: 納品先E2Eテスト — 実装計画

## 概要

`feat/issue-240` ブランチで実装された納品先管理画面（一覧・詳細）のE2Eテストを作成する。
既存の得意先テスト（`customers-list.e2e.ts`, `customers-detail.e2e.ts`）と同じパターンに従い、
検索・表示・ナビゲーション・NULLハンドリング・404 を網羅する。

テストに必要な無効状態の納品先がシードデータに存在しないため、D007（無効）を追加する。

## 設計判断

### 得意先ドロップダウン検索の検証方式
- A. ドロップダウンのoption要素からUUIDを取得し、URLパラメータも完全検証
- B. テキスト（得意先名）で選択し、表示結果のみで検証
- 採用: B（UUIDは動的生成されるため、テストの安定性を優先）

### D007の配置先
- C003（大阪機械工業株式会社）配下に追加
- 理由: C001詳細テストは配下2件を期待、C004詳細テストは0件を期待しているため、既存テストに影響しない

### Step 2: 納品先一覧E2Eテスト作成

  - `test.describe("納品先一覧（管理者）")`:
