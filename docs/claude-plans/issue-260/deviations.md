# Issue #260 実装の逸脱記録

## 1. 「周辺商品追加」テストの扱い

### 元の計画 / Issue 文言

> - [ ] 「周辺商品追加」テストを `products-relations.e2e.ts` に移動

Issue 文言は crud の「管理者が周辺商品を設定できる」テストを relations ファイルへ
**移動（コピー）** する想定。

### 実際の実装

crud から当該テストを **削除**（relations へのコピーはしない）。

### 逸脱の理由

`products-relations.e2e.ts` の既存テスト「商品コードで検索して商品を選択・追加・
保存できる」が、crud の「周辺商品を設定できる」を **上位互換でカバー** していた
（モーダル検索→選択→追加→保存→永続化に加え、自己除外・件数ボタン活性・数量
デフォルト値も検証）。literal にコピーすると重複テストが生じ、skill §4（関心事の
集約）と「完了条件: relations 機能のテストが products-relations.e2e.ts に集約」に
反する。よって crud からは削除し、relations 側の既存カバレッジで担保する方針とした。

## 2. 一般ユーザー簡易 chain（§11）の省略

### 元の計画 / Issue 文言

> - [ ] 一般ユーザー簡易 chain の要否判断・必要なら追加

### 実際の実装

簡易 chain（作成→削除）は **追加せず省略**。代わりにサーバ認可回帰テスト
「一般ユーザーは商品を作成できない（サーバ認可で拒否）」を追加。

### 逸脱の理由（＝要否判断の結論）

商品作成 server action `createProduct` が `verifyAdmin()` を必須とし、一般ユーザーは
そもそも商品を作成できない（`/signin?reason=forbidden` へ redirect）。よって §11 の
一般ユーザー簡易 chain は **適用不可**。`employees-crud.e2e.ts`（#266）の前例に準拠し、
省略根拠をコード内コメントに明記したうえで省略した。
