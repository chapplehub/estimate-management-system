# Issue #481: 共通販売単価 保守画面の E2E テスト作成 — 実装計画

## 概要

共通販売単価 保守画面（一覧 UC-1／詳細 UC-2／登録 UC-3／編集・適用終了 UC-4／削除 UC-5）の E2E テストを作成する。実BE接続済み（#473/#480）の現状画面を対象とし、後続の #474 改定ウィザード・#475 タイムライン追加や BE 変更に対する回帰の安全網を張る。

ADR-0012 / 0017 / 0020 と `create-e2e-test` スキルに準拠する。時点依存（状態が `new Date()` 由来）は ADR-20260629-3x5 の today 相対シード方針で扱う。

**前提（#481 着手前に完了させる）**
1. **#482** 権限ボタン出し分け（一般ユーザーにミューテーション系ボタンを非表示にする）。
2. **#483** 表示語を正準語「共通販売単価」へ統一（E2E は「共通販売単価」をアサートする）。

## 設計判断

### テスト対象スコープ
- UC-1〜UC-5 の全フロー＋ドメインエラー（重複拒否）＋権限を対象とする。
- #474 改定ウィザード・#475 タイムラインは未実装のため対象外。
- 理由: 実BE接続済みで動く範囲に安全網を張るのが本Issueの目的。未実装画面には張れない。

### 時点依存の扱い（→ ADR-20260629-3x5）
- A. 基準日の env 注入機構を本番に追加 / B. today 相対シード＋JST 起点ヘルパ / C. 固定日付で単調状態のみ検証
- **採用: B**。理由: 本番非侵入（ADR-0017 と同じ精神）・実行日に依らず決定的・将来行必須の UC-4/5 まで網羅できる。

### 権限テストの形
- 一般ユーザー: ボタン非表示（#482 後）＋一覧/詳細は閲覧可、をアサート。#482（PR #484）の実装は一般ユーザーに対し新規追加ボタン・各行の操作ボタンに加え「操作」列ヘッダー（`<th>操作</th>`）ごと描画しない。よって「操作」列ヘッダーの非在をアサートする（ボタン個別より堅牢）。
- サーバ側 `verifyAdmin` リダイレクトは **E2E 対象外**。理由: #482 後は UI から不可達（admin 専用ページが無く、ページは全ユーザー閲覧可）。Server Action 直接 POST は Next 内部仕様に密結合で脆い。認可の正本検証は BE/アクション層に委ね、`deviations.md` に ADR-0020 からの逸脱として記録する。

### ファイル構成（3ファイル・ADR-0020 判断4）
- 詳細画面に戻りリンク・404・厚い状態別表示があるため list / detail / crud の3分割。

### 直列 chain の分割（ADR-0020 判断1）
- Chain A（PRD903・将来行ライフサイクル）と Chain B（PRD905・適用終了/改定）を別 chain にする。理由: 関心事（ライフサイクル vs end-dating）とデータが異なる。各3手で blast radius も小さい。

### シードフィクスチャ（PRD9NN 帯・ADR-0020 判断2 / ADR-20260629-3x5）
- 共通シード（並列・DB不変）: PRD901（失効/現在有効/将来の3状態）・PRD902（未設定）・PRD904（失効のみ）。
- CRUD（直列・期間はテスト内生成）: PRD903（登録→編集→削除）・PRD905（現在有効登録→適用終了→新将来登録）。
- 過去開始（expired）は集約の `assertStartNotPast` を通せないため raw daterange insert で投入。
- 楽観ロック競合・状態別サーバ拒否は E2E 対象外（誘発困難＋#482 後はUIから不可達）。

## ステップ

### Step 1: E2E シードに共通販売単価フィクスチャを追加
- 対象ファイル: `prisma/seed-e2e.ts`
- 作業内容:
  - PRD901/902/903/904/905（INDIVIDUAL・isActive=true）を PRODUCTS に追加。
  - JST today 起点の相対日付ヘルパを用意（`toJstCalendarDay` と同じ +9h ロジック）。
  - PRD901: 失効`[today-60,today-30)`¥1000 / 現在有効`[today-30,today+30)`¥2000 / 将来`[today+30,null)`¥3000 を raw daterange insert。
  - PRD904: 失効`[today-60,today-30)` を raw insert。
  - PRD902/903/905 は CSP 集約を作らない（未設定で開始）。
- コミットメッセージ: `test: 共通販売単価 E2E シード（PRD9NN・today相対の3状態/未設定/失効）を追加`
