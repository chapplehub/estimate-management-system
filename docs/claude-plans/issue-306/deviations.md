# Issue #306: 実装計画からの逸脱記録

## 1. Step 6（テスト）を独立コミットにせず、各実装コミットへ織り込んだ

- **元の計画内容**: Step 6 として「リポジトリ統合テスト + コマンド単体テスト」を最後にまとめて追加し、`test:` コミットとして独立させる。
- **実際の実装内容**: `/tdd`（red-green-refactor）で進めたため、各テストは対応する実装と同じコミットに含まれる（Step 1 のコミットに insert/update/競合の統合テスト、Step 2 のコミットに細目7の統合テスト、Step 3 のコミットに6コマンドの競合テスト）。テストの内容自体は計画どおり（細目7シナリオ含む）。
- **逸脱の理由**: TDD ではテストが実装を駆動するため、テストだけを後置きする計画のステップ分割と構造的に両立しない。「意味のあるまとまりでコミットする」規約に照らし、テストと実装を同一コミットに置く方が整合的と判断した。
