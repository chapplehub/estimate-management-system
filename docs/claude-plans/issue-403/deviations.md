# Issue #403 実装 — 計画からの逸脱記録

## 逸脱1: 異通貨比較の異常系テストを追加せず、コメントでの明記に変更

- **元の計画内容**: Step 1 のテストに「異常系（異通貨で `InvalidArgumentError` が比較専用メッセージで送出）」を追加する。
- **実際の実装内容**: 異通貨比較の異常系テストは追加せず、既存テストの注記コメント（`Money.test.ts` 異常系末尾）を「演算・比較の両方を第2通貨追加時にテストする」と更新するに留めた。
- **逸脱の理由**: `Currency` クラスは private constructor で `JPY` のみを公開しており、テストから第2の通貨を構築できない（異通貨経路は到達不能）。これは既存の `add`/`subtract` 異通貨テストと全く同じ制約で、既存コードもテストではなくコメントで制約を明記する方針を取っていた。到達不能なテストを偽装するより、既存の方針に揃えるのが誠実と判断した。`assertComparableCurrency` のロジックは `assertSameCurrency` と同型のため、レビューで担保される。第2通貨追加時に演算・比較の両方のテストを足す旨をコメントに残した。
