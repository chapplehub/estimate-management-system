# Issue #468 原価移設 B — 実装中の計画逸脱記録

計画ファイル: `plan.md`（ADR-20260627-a5c 改訂版 準拠）。計画と異なる対応をした点を記録する。

## 1. スコープを「画面付け替え」から「純粋契約（撤去）」へ変更（最重要）

- **元の計画（ADR-20260627-a5c 初版 Issue B）**: Product 作成/編集画面の原価欄を新原価集約へ
  付け替え（2集約オーケストレーション・二重楽観ロック）。回帰窓（原価が編集不能になる隙）を
  即時に閉じる。
- **実際の実装**: 原価UI・読み取り経路・Product 集約からの原価・`products.cost_price` 列・旧VO を
  **すべて撤去する純減算**にとどめ、新集約への書き込み配線は #468 に持ち込まない。原価の編集・
  閲覧の再配線は後続の専用原価管理画面 **#470** へ送る。
- **逸脱の理由**: 時間有効性を持つ原価集約（append-only 期間タイムライン）を単一値フォームへ写す
  インピーダンスミスマッチ（現行開放端期間の上書き／close-and-open／close-period のリポジトリ追加）
  を #468 に持ち込むと過剰実装になる。原価タイムラインの消費者（粗利接続）も未存在。pre-production
  （開発段階・ユーザ無し）ゆえ #468 完了〜#470 配線間の編集・閲覧不能を受容できる。ADR は本ブランチで
  in-place 改訂済（コミット `27e695a`）。後続イシュー #470 を起票済。

## 2. 書き込み経路の除去を1コミットに集約（計画の Step 1/Step 3 を統合）

- **元の計画**: Step 1（domain＋application＋infra-write）と Step 3（画面UI）を別コミット。
- **実際の実装**: 画面フォーム/Server Action → コマンド → Product 集約 → Mapper → 旧VO削除を
  **1コミット**（`fd4f15e`）にまとめた。
- **逸脱の理由**: コマンド入力から `costPrice` を外すと、それを渡す Server Action が即型エラーになり、
  レイヤー単位の分割では各コミットが型割れで pre-commit（lint-staged＋vitest）を通らない。密結合
  ゆえ書き込み経路は1単位が green を保てる。読み取り経路（`0218cc0`）と列削除（`23a195d`）は
  独立に分離できたため計画どおり別コミット。

## 3. 計画に未記載だった除去対象を追加で処理

- **元の計画**: 主要ファイルのみ列挙。
- **実際の実装**: 旧VOの**テスト** `product/domain/values/__tests__/CostPrice.test.ts` の削除、
  `Product.reconstruct` を旧シグネチャ（13引数）で呼ぶ3テスト（Activate/Deactivate/
  ProductReplacement）の修正、読み取り系テストヘルパ（Search/GetProductById の
  `prisma.product.create` 経由 costPrice 投入）の除去を追加で行った。
- **逸脱の理由**: tsc による横断検出で発覚。計画の網羅漏れを実体に合わせて補完した。
