# Issue #360 実装の逸脱記録

CLAUDE.md「Record deviations from plan」に基づき、計画・issue 本文からの逸脱を記録する。

## 逸脱1: issue 本文の「#351 作成画面に統合」前提を退けた（ADR-0057）

- **元の計画（issue 本文）**: 見積複製(C6) の UI を #351 の C1 新規作成画面に「ヘッダ引き継ぎ＋
  バリ選択プリフィルの新規作成」として統合する。根拠として ADR-0042 の「見積のみ複製＝プレゼン層の
  ヘッダ引き継ぎ新規作成 UX」を挙げていた。
- **実際の実装**: 複製元詳細画面のヘッダー操作エリアに「見積複製」ボタンを新設し、モーダルで
  バリ選択(≥1)・見積日・締切日・部署を収集して `DuplicateEstimateCommand` を駆動、新採番の別見積
  詳細画面へ遷移する。#351 作成画面には手を入れず、共有部品（`resolveEffectiveTaxRate`・日付
  ユーティリティ・`REDIRECT_REASON` パターン・部署 select・`isVariationDuplicatable`）の再利用に縮小した。
- **逸脱の理由**: C6 は C1 `CreateEstimate` とは駆動コマンド・入力形状・成果物（複製系譜
  `EstimateVariationCopy` の記録・見積単価のクリア）が異なり、作成画面統合では #360 の受け入れ条件
  （≥1選択・系譜記録・単価クリア）を満たせない。ADR-0042 の「ヘッダ引き継ぎ新規作成」は §5.2 の
  バリ0件ケース（系譜なしの将来 UX）であり、≥1選択・系譜ありの C6 とは別物。issue 本文はこの2つを
  混同していた。詳細は ADR-0057 を参照。

## 逸脱2: 実装ステップ Step3（モーダル）と Step4（Server Action）の順序を入れ替えた

- **元の計画**: Step3 で複製モーダル本体、Step4 で Server Action ＋ボタン配線。
- **実際の実装**: 先に Server Action ＋ `REDIRECT_REASON.ESTIMATE_DUPLICATED` ＋フラッシュ（サーバ側）を
  実装し、次にモーダル本体＋ヘッダーボタン配線（クライアント側）を実装した。
- **逸脱の理由**: モーダルは `useServerForm` で `duplicateEstimate` アクションを束ねるため、アクションが
  先に存在しないとクライアント側コミットが単体でコンパイルできない。各コミットがビルド可能であることを
  優先し、コンパイル依存の自然な順序に並べ替えた。設計判断の変更ではなく実装順のみの調整。
