# issue #333（S5 セット明細の編集）— 計画からの逸脱記録

`/grill-with-docs` セッション（2026-06-16）で確定した、ドキュメント済み計画からの逸脱。

## 1. 集約越え検証: ドメインポート → ADR-0030（メソッド引数）

- **元の計画**: ADR-0047 補足が「ネスト禁止・構成商品は個別/消耗品のみ、を `estimate/domain/services/` のルール検証型 Domain Service を**ドメインポート経由**で実装（実装は S5）」と前約束。
- **実際の対応**: ドメインポートは新設せず、**ADR-0030（pass-cross-cutting-context-via-method-arguments）** に従い、アプリ層が `ProductQueryService` で区分・`isActive` をライブ取得 → 純粋ドメイン検証へメソッド引数で渡す。新規 ADR-0052 として記録。
- **逸脱の理由**: ADR-0047 補足は ADR-0030 を踏まえていなかった。リポジトリには集約越えドメインポートの前例が皆無で、ポートは ADR-0030 と非対称な新パターンを増やす。同じ目的を既存方針で達成できる。

## 2. 区分検証のハード違反が到達する経路＝手動構成追加を繰り延べ

- **元の計画**: 行改良指示書 §3 / §4.3 は「構成明細をアクティブにして明細追加で同じ群へ構成商品を手動追加」できる補完操作を想定。
- **実際の対応**: **手動構成追加は S5 スコープ外**とし issue **#350** へ繰り延べ。S5 のセット群は**自動展開のみ**で形成（構成は数量・単価・メモ編集と削除は可能、手動追加は不可）。区分/ネスト違反は自動展開では構造的に到達不能になるため、ADR-0052 の検証は**ペイロード防御**として導入する。
- **逸脱の理由**: issue #333 の受け入れ条件に手動追加が無い。自動展開だけなら違反は到達不能で、スライスを 1 つに収められる（スコープ肥大の回避）。

## 3. インライン商品差し替え UI は不採用 → §8 残課題は該当なし

- **元の計画**: 行改良指示書 §4.3 は「行の商品選択コンボボックスでセット商品を選んだら set-parent へ切替」を想定し、§8 残課題に「通常商品→セット商品への切替時の既存値（メモ・数量）の扱い」を挙げていた。
- **実際の対応**: S4 が既に**インライン商品差し替えを実装していない**（商品は追加時スナップショット固定・差し替え不可）。S5 もこれを踏襲し、SET は**明細追加モーダルから新規ノードとして追加**する。切替操作が存在しないため、**§8 残課題「切替時の既存値の扱い」は該当なし**として閉じる。
- **逸脱の理由**: 残課題が前提とする「インライン商品差し替え」操作を S4 が採用していないため、残課題そのものが消滅する。

## 4. C1（create 経路）の配線スコープ

- **元の計画**: issue #333 タスクに「C1/C4 のセット書き込み配線」。
- **実際の対応**: C1 `CreateEstimate` は analysis 01 のとおり**詳細画面の対象外（新規作成は別画面）**。S5 では C1 を **app/factory/domain 層まで**配線（`EstimateVariationDescriptor` / `buildVariation` に setGroups を通す）するが、**create 画面 UI は作らない**（issue **#351** へ）。S5 の UI は C4（`VariationEditForm`）のみ。
- **逸脱の理由**: create 画面は未実装の別スライス。ドメイン/ファクトリの対称性は保ちつつ（seed/テストでセット付き見積を生成可能）、UI は別 issue に切り出す。

## 補足: 逸脱ではない継続事項

- **JSON 往復の入れ子化（採用 (A)）**: 作業コピー／JSON を `(LineNode | SetGroupNode)[]` のトップレベル判別子 union とし、`SetGroupNode` が `components` を内包。これは ADR-0050 が「S5 も同じ往復方式を継承・セット群表現を JSON に拡張する想定」と明記した路線の具体化であり、逸脱ではない。
- **`LineDTO` への `isActive` 追加**: 無効構成の警告（インライン・状態導出）のため、read クエリが既に join 済みの Product 行から `isActive` を 1 列写す（ADR-0048 read-through の延長）。
