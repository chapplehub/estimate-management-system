# 共通売単価 保守画面 — プロトタイプ→FEコンポーネント変換 設計合意（#429 第一段階）

> claude.ai design のプロトタイプ（`docs/design/common-selling-price-maintenance/`）を、本プロジェクト（Next.js App Router / React 19 / Tailwind / DDD）の規約に合わせて実装するための設計合意。`grill-with-docs` セッションで確定。用語の正準は `CONTEXT.md`「価格」節、ユースケースは `docs/claude-plans/issue-429/use-cases.md`。

## 確定した設計判断

| # | 論点 | 決定 | 根拠 |
|---|---|---|---|
| 1 | 変換ゴール | **プレゼンテーション層先行**（モック境界、#466完成後に中身差し替え） | #429はFE画面のみに純化済み。BE基盤は#466待ち |
| 2 | ルーティング | **ページ分割**: `/common-selling-prices`（一覧）＋`/common-selling-prices/[productCd]`（明細）。フォームは詳細内パネル | 既存マスタ全画面の規約。集約境界＝商品単位URLと一致 |
| 3 | 書き込みモック | **インメモリ・ストア**（Action は parse→ミューテート→revalidate の本番同形） | 状態遷移をクリックで検証。Action のガワを本番同形に保つ |
| 4 | 表示モード | **表（table）のみ**。timeline は後回し | UC-1〜5は表で完結。timelineは付加的可視化で後付け可 |
| 5 | 改訂 | **ウィザード後回し**。適用終了の編集／期間追加の2原子操作を統一フォームで先に固める | ウィザードは暫定イメージ＋迂回バグあり。原子操作の合成糖衣 |
| 6 | フォーム統一 | **単一 `PeriodForm`＋状態別フィールドロック**（現在有効＝終了日のみ活性） | 「適用終了は編集の一形態」(use-cases)。#351の統一方針 |
| 7 | フォーム表示 | **インラインパネル**（詳細ページ内展開） | プロトタイプ準拠。状態ロックを対象行の文脈で表示 |
| 8 | 一覧 | `SearchForm`＋`DataTable`再利用。列＝コード/名/現在有効単価(未設定・失効中バッジ)。絞り込み＝コード・名・未設定のみ | UC-1の確定要求を過不足なく満たす |
| 9 | 楽観ロック | **version 往復＋Action でチェック＆bump**（ADR-0039 同形） | 画面側のフォーム/Action/DTO形がversion前提で決まる |
| 10 | 刻み | **1ラウンド目＝データ境界＋一覧画面（UC-1）**。詳細・フォームは次ラウンド | 記憶「1回1画面」。DTO形を一覧で実地検証してから詳細へ |

## モック境界の構造（#466接続時の差し替え点）

- 配置: `src/app/(features)/common-selling-prices/_data/`
  - `types.ts` — 読みモデルDTO（一覧アイテム / 期間明細 / version付き集約明細）。**#466の読みモデル仕様に逆フィード**する。
  - `mock-store.ts` — インメモリ・ストア（プロトタイプのシード PRD001〜015 を流用）。**モック専用・差し替え対象**。
  - `queries.ts` — 将来の QueryService と同形の async fetch 関数（`fetchCommonSellingPriceList` / `fetchCommonSellingPriceDetail`）。**#466完成後は中身だけ Factory→QueryService に差し替え**、シグネチャは維持。
- 参照日: `REFERENCE_DATE`（`2026-06-27`、プロトタイプ準拠）を1箇所に集約。#466未決の「参照日の注入方法」をFE側でも1点に閉じ込め、将来クロック/引数注入へ差し替え可能にする。

## 用語・ADR

- CONTEXT.md は「適用終了/失効/無効化」を既に網羅。今回 新規の正準用語は無し（追記不要）。
- 上記はいずれも**可逆**（#466接続で差し替え）かつ既存規約準拠のため、ADRは起こさない。

## ラウンド構成

- **ラウンド1（本タスク）**: ①データ境界（types / mock-store / queries）→ ②一覧画面（UC-1）。各まとまりでコミット。
- **ラウンド2（次回）**: 詳細ページ（期間明細・状態バッジ UC-2）→ `PeriodForm`（追加 UC-3 / 将来行編集・現在有効行の適用終了 UC-4 / 削除 UC-5）→ 楽観ロック経路。
