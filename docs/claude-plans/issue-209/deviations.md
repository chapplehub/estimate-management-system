# 計画からの逸脱記録

## 逸脱 1: disabled select に hidden input が必要
- **計画**: Step 5「商品区分は disabled select で表示のみ」
- **実際**: disabled select は HTML 仕様で FormData に値が含まれないため、Conform のクライアントバリデーション（Zod の category 必須チェック）が失敗し、フォーム送信がブロックされた。`<input type="hidden">` を追加して解決
- **理由**: 計画時に disabled 要素の FormData 除外仕様を見落としていた

## 逸脱 2: h1/h2 見出し規約の考慮漏れ → Step 10 後に追加修正
- **計画**: 各ページの見出しについて命名規約の言及なし。暗黙的にページ固有の名前（h1="商品詳細"、h1="商品編集" 等）で実装
- **実際**: 従業員・部署画面では h1="○○管理"、h2="○○詳細/○○変更" という規約があり、商品画面でも同じ規約に合わせる必要があった。規約に合わせなかったことで、h1 と h2 に同じテキストが入るケースが発生し、Playwright の `getByRole("heading")` で strict mode violation が発生。Step 10 の後に全商品ページの h1 を「商品管理」に統一し、h2 を各ページ固有の名前に修正
- **理由**: 計画作成時に既存画面の見出し規約を調査していなかった。E2E テストのセレクタも `getByText` から `dt + dd` ラベルペアパターンに改善
