# Issue #238: 得意先一覧・詳細画面のE2Eテスト — 実装計画

## 概要

得意先一覧画面（`/customers`）と得意先詳細画面（`/customers/[code]`）のE2Eテストを作成する。
一覧画面では全9検索条件の個別テスト＋複合条件テストを実施。詳細画面では基本情報・納品先表示・エッジケースをカバーする。
テストデータが不足しているため、seed-e2e.ts に得意先2件を追加する。

## 設計判断

### シードデータの追加方針
- 既存3社はすべて `isActive: true` → 状態フィルタのテストに `isActive: false` の得意先が必要
- 既存3社はすべてフィールドが埋まっている → NULL表示テスト用に一部NULL項目のある得意先が必要
- 推奨: C004（無効・納品先なし）とC005（NULL項目あり）の2件を追加

### 納品先リンクの遷移テスト
- `/delivery-locations/[code]` ページはまだ未実装
- 推奨: リンクの `href` 属性を検証するのみ（実際のクリック遷移は行わない）

## ステップ

### Step 1: シードデータ追加
- 対象ファイル: `prisma/seed-e2e.ts`
- 作業内容:
  - CUSTOMERS配列の既存3件に `isActive: true` フィールドを明示的に追加
  - C004を追加: 名古屋精密機器株式会社（愛知県、`isActive: false`、納品先なし）
  - C005を追加: 福岡商事株式会社（福岡県、`faxNumber: null`、`contactPerson: null`、`marginRate: null`）
  - `seedCustomersAndDeliveryLocations()` 内の `isActive` を `customerData.isActive ?? true` に変更（現在ハードコード `true`）
- コミットメッセージ: `test: E2Eテスト用の得意先シードデータを追加`

### Step 2: 得意先一覧E2Eテスト作成
- 対象ファイル: `src/app/(features)/customers/customers-list.e2e.ts`（新規作成）
- 作業内容:
  - `waitForListReady()` ヘルパー（見出し「得意先管理」＋テーブル行の表示待ち）
  - 管理者テスト（`test.describe("得意先一覧（管理者）")`）:
    1. 一覧表示の確認（「新規登録」ボタンは存在しないことも確認）
    2. 名前で検索（部分一致）— "山田" → C001のみ
    3. コードで検索（完全一致）— "C001" → 1件
    4. 郵便番号で検索（完全一致）— "1000001" → C001
    5. 都道府県で検索 — "東京都" → 2件（C001, C002）、全行の都道府県列が「東京都」
    6. 住所で検索（部分一致）— "梅田" → C003のみ
    7. 電話番号で検索（完全一致）— "0312345678" → C001
    8. FAX番号で検索（完全一致）— "0661234568" → C003
    9. 担当者で検索（部分一致）— "田中" → C003のみ
    10. 状態「有効」で検索 → 4件、全行のステータスバッジが「有効」
    11. 状態「無効」で検索 → C004のみ、ステータスバッジが「無効」
    12. 複合条件検索 — 名前"電子" ＋ 都道府県"東京都" → C002のみ
    13. クリアボタンで検索条件リセット
    14. コードリンクから詳細画面に遷移
  - 一般ユーザーテスト（`test.describe("得意先一覧（一般ユーザー）")`）:
    15. 一般ユーザーでも一覧を閲覧できる
- コミットメッセージ: `test: 得意先一覧画面のE2Eテストを追加`

### Step 3: 得意先詳細E2Eテスト作成
- 対象ファイル: `src/app/(features)/customers/customers-detail.e2e.ts`（新規作成）
- 作業内容:
  - テストケース:
    1. 基本情報が正しく表示される（C001: コード、名前、郵便番号フォーマット、都道府県、住所、電話、FAX、担当者、状態バッジ）

    3. パンくずリンクから一覧に戻れる
