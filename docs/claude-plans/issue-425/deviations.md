# Issue #425 実装の計画からの逸脱

## 1. Step 1（ドメイン層）と Step 2（アプリ層）を1コミットに統合

- **元の計画**: Step 1「ドメイン層から marginRate を削除」と Step 2「アプリ層の追従」をそれぞれ独立したコミットにする。
- **実際の実装**: 両者を1コミット（`refactor: customer ドメイン〜アプリ層から marginRate を削除`）に統合した。
- **逸脱の理由**: `Customer.reconstruct` は位置引数で、marginRate を抜くと Mapper の呼び出し側が即座にコンパイル不能になる。ドメイン層だけを切り出してコミットすると「ビルドの通らないスナップショット」が履歴に残り、合意した「コミット毎グリーンバー」原則に反する。両層は reconstruct シグネチャで密結合しているため、最小のグリーン単位として統合した。

## 2. 受け入れ条件の grep スコープ修正（計画段階で合意済み・記録目的）

- **元の Issue 記載**: `grep -rni "margin" src prisma` = 0 ヒット。
- **実際**: `grep -rni "margin" src prisma --exclude-dir=migrations` = 0 ヒットへ修正。
- **理由**: マイグレーション履歴 SQL（`20260217074132`, `20260415012955`, および本件の `20260623002537`）は適用済みの不変記録で `margin_rate` を含む。履歴を書き換えると整合性が壊れるため、grep スコープから migrations を除外した。（grill-with-docs セッションで合意済み。逸脱というより条件の精緻化。）
